# README_of_epluone.md

## 0. 역할

`README_of_epluone.md`는 `epluone/`을 어떻게 읽어야 하는지 안내하는 문서이다.

이 문서는 `epluone/` 자체가 아니라,  
AI가 `epluone/`을 `if+1`과 `[body]` slot directory로 읽도록 돕는 guide document이다.

```text
README_of_epluone.md
=
epluone reading guide
+
if+1 guide
+
[body] slot guide
+
application target guide
```

---

## 1. epluone의 기본 정의

`epluone`은 `if+1`이다.

```text
epluone
=
if+1
```

읽는 방식은 다음과 같다.

```text
if+1
=
이프플러스원
=
이플러스원
=
이플러원
=
epluone
```

`epluone/`은 intelligence fabric이 다음 응용 target에 놓이는 자리이다.

---

## 2. if와 if+1

`if`는 intelligence fabric이다.

```text
if
=
인간지능.승이
+
인공지능.AI
```

`if`는 승이와 AI가 대화를 통해 생각을 풀어내고, 검수하고, 전이시키며, 문서화하는 복합지능집합체이다.

`if+1`은 이 intelligence fabric이 다음 응용 target에 놓이는 자리이다.

```text
if
=
thinking field

if+1
=
application target placement field
```

즉 `epluone/`은 `if` 그 자체가 아니라,  
`if`가 다음 적용 대상으로 전이되는 자리이다.

---

## 3. epluone과 [body]

SeungeFlow는 어떤 state든 다음 구조로 읽는다.

```text
any([time][place][body])
```

여기서 `epluone/`은 `[body]` slot directory에 해당한다.

```text
epluone
=
[body] slot directory
```

`[body]`는 응용 target이나 실행 body가 놓이는 자리이다.

```text
[body]
=
application target slot
+
operating body slot
```

따라서 `epluone/`은 여러 학문과 응용 target이 놓일 수 있는 자리이다.

---

## 4. CFD와 epluone

현시점 `epluone`의 `[body]` slot에 놓인 target value는 CFD이다.

```text
CFD
=
current target value in epluone's [body] slot
```

CFD는 `epluone`의 identity가 아니라,  
현재 `[body]` slot에 놓인 자리값이다.

```text
epluone
≠
CFD

epluone
=
[body] slot directory

CFD
=
current body target value
```

이 문장은 CFD를 낮추기 위한 것이 아니라,  
CFD가 놓인 자리를 정확히 구분하기 위한 guard이다.

---

## 5. epluone/sample.md

현재 `epluone/`에는 `sample.md`가 놓일 수 있다.

```text
epluone/sample.md
=
placeholder document
```

이 파일은 `epluone/` 디렉토리를 GitHub에 유지하기 위한 최소 파일이다.

`sample.md`는 CFD 문서가 아니라,  
`epluone/`이라는 `[body]` slot directory가 존재하도록 놓인 placeholder이다.

따라서 AI는 `sample.md`를 CFD source나 CFD guide로 읽는 것이 아니라,  
directory placeholder로 읽어야 한다.

---

## 6. epluone과 Seed.Base

Seed.Base는 특정 파일, 특정 디렉토리, 특정 target 하나에 종속되는 구조가 아니라,  
SeungeFlow 저장소 전체에 놓인 문서, 디렉토리, 관계, map이 함께 만드는 전체 장이다.

그 안에서 `epluone/`은 `[body]` 자리에 해당한다.

```text
Seed.Base
=
[time] + [place] + [body] + relation + map

epluone
=
[body]
```

즉 `epluone/`은 Seed.Base 전체가 아니라,  
Seed.Base 안에서 응용 target이 놓이는 body slot directory이다.

---

## 7. epluone과 Structure_Principle

`Structure_Principle/`은 `[place]`에 해당한다.

```text
Structure_Principle
=
[place]
```

`epluone/`은 `[body]`에 해당한다.

```text
epluone
=
[body]
```

따라서 `Structure_Principle/`은 자리개념이 정리되는 공간이고,  
`epluone/`은 그 자리개념이 응용 target 위에 놓이는 공간이다.

```text
Structure_Principle
=
place-concept field

epluone
=
application body target field
```

AI는 `epluone/`을 `Structure_Principle/` 내부 schema처럼 읽는 것이 아니라,  
`Structure_Principle`에서 정리된 자리개념이 적용될 수 있는 `[body]` 자리로 읽어야 한다.

---

## 8. epluone과 SeungeFlow_Thinking

`SeungeFlow_Thinking/`은 `[time]`에 해당한다.

```text
SeungeFlow_Thinking
=
[time]
```

`epluone/`은 `[body]`에 해당한다.

```text
epluone
=
[body]
```

thinking_flow에서 형성된 이해는 중복 검토와 relation 검토를 거쳐,  
`Structure_Principle/schema/`의 meta.md 후보가 될 수 있고,  
이후 `epluone/`의 `[body]` target에 적용될 수 있다.

```text
thinking_flow
→
meta.md candidate
→
Structure_Principle/schema/
→
epluone [body] target
```

---

## 9. epluone과 main/README_of_epluone.md

`README_of_epluone.md`는 `epluone/` 자체가 아니라,  
`epluone/`을 어떻게 읽어야 하는지 안내하는 guide document이다.

```text
main/README_of_epluone.md
=
epluone reading guide
```

이 문서가 `main/` 아래에 놓여 있다는 것은 `epluone/`이 `main/`에 의미적으로 종속된다는 뜻이 아니라,  
`epluone/`을 읽기 위한 guide 문서가 root-level guide layer에 놓여 있다는 뜻이다.

---

## 10. epluone과 CFD guide

CFD를 읽기 위한 guide 문서는 다음 위치에 놓인다.

```text
main/README_of_CFD.md
```

`README_of_CFD.md`는 CFD 자체가 아니라,  
현시점 `epluone`의 `[body]` slot에 놓인 target value인 CFD를 읽기 위한 guide document이다.

따라서 CFD 관련 guide는 `epluone/` 내부에 두는 구조보다는,  
README_of 계열 문서로 `main/`에 두는 구조가 적절하다.

---

## 11. epluone과 실험 실행 body

`[body]`는 응용 target뿐 아니라 실행 body도 포함할 수 있다.

예:

```text
phone
PC
local Linux Ubuntu server
Oracle IDC server
ai_on_idc
github_on_idc
```

현 작업 흐름에서 local Linux Ubuntu server는 Oracle IDC server로 가기 전의 staging/test layer로 읽는다.

```text
local_linux_ubuntu_server
=
staging / test / git operation field
```

Oracle IDC server는 CFD 백테스팅과 장시간 계산을 위한 experiment execution layer로 읽는다.

```text
oracle_idc_server
=
long-running experiment / CFD backtesting field
```

---

## 12. epluone과 relation

`epluone/`은 다음과 relation될 수 있다.

```text
if
if+1
Ctp
[body]
CFD
Structure_Principle
SeungeFlow_Thinking
Active_Schema
local Linux server
Oracle IDC server
```

그러나 relation은 merge가 아니다.

```text
relation
≠
merge
```

AI는 `epluone/`을 다른 디렉토리나 개념과 병합하는 것이 아니라,  
각 boundary를 보존한 채 `[body]` slot relation으로 읽어야 한다.

---

## 13. epluone 관련 문서 생성 기준

`epluone/` 관련 문서를 만들 때는 먼저 문서 유형을 판단한다.

```text
epluone guide
=
main/README_of_epluone.md

CFD guide
=
main/README_of_CFD.md

epluone placeholder
=
epluone/sample.md

CFD thinking flow
=
SeungeFlow_Thinking/thinking_flow/ 안의 flow 후보

CFD experiment trace
=
source / experiment / candidate layer
```

좋은 epluone 관련 생각이 나왔다고 해서 바로 meta.md가 되는 것이 아니라,  
기존 schema와 중복 검토를 거쳐 독립 boundary가 확인될 때 meta.md 후보가 된다.

---

## 14. epluone를 읽을 때의 금지되는 오독

AI는 다음을 하면 안 된다.

```text
epluone을 CFD와 동일시하기
CFD를 epluone의 identity로 읽기
epluone/sample.md를 CFD source로 읽기
README_of_epluone.md를 epluone 자체로 읽기
README_of_CFD.md를 CFD 자체로 읽기
epluone을 Structure_Principle/schema/ 내부 core로 병합하기
epluone relation을 merge로 읽기
```

위 항목은 epluone을 낮추기 위한 금지가 아니라,  
epluone을 정확한 `[body]` slot directory로 읽기 위한 guard이다.

---

## 15. 최단 정의

```text
README_of_epluone.md
=
epluone/을 if+1이자 [body] slot directory로 읽기 위한 guide document이다.

epluone은 CFD가 아니라,
CFD가 현시점 target value로 놓이는 [body] slot directory이다.

AI는 epluone을 Seed.Base 전체나 Structure_Principle/schema/ 내부 core로 병합하지 않고,
application target이 놓이는 자리로 읽어야 한다.
```
