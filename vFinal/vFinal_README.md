## SEPARATION PRINCIPLE

The files in `vFinal` are not a single uniform manuscript.

They must be separated by role.

Separation is defined by the following rule:

```txt
1. Main Structural Execution Line
2. Integrated Accumulation Marker Line
3. Final Accumulation Declaration Line
4. RH Specialized Kernel Line
5. Repository Definition Line
```

These lines must remain distinguishable.

## INTERPRETATION RULE

When reading `vFinal`, the following order must be preserved:

```txt
1. repository definition
2. main execution line
3. integrated accumulation marker
4. final accumulation declaration
5. RH specialized kernel
```

This prevents line-mixing.

This prevents structural noise.

This prevents false equivalence between:

- execution records
- accumulation declarations
- RH-specialized kernel files
- structure essence README

---

## NON-MERGE RULE

The following merges are forbidden at the definition level:

```txt
1. README.md = vFinal
2. vFinal_v1.00.md = entire vFinal
3. vFinal_v0.80.md = full integrated archive
4. vFinal_0021.md / vFinal_0031.md = main structural execution line
```

Reason:

These files do not have the same role.
