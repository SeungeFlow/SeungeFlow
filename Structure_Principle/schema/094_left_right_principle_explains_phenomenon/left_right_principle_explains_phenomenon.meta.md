# META: left_right_principle_explains_phenomenon

## role

left_right_principle_explains_phenomenon은 왼쪽자리에 있는 원리층이 오른쪽자리에 드러난 현상층을 설명하는 구조를 정의하는 schema이다.

## core

```text
왼쪽 = 원리
오른쪽 = 현상
왼쪽자리에 있는 것이 오른쪽자리에 있는 것을 설명한다.
```

## definition

오른쪽에 보이는 것은 현상이다.

왼쪽에 있는 것은 그 현상을 발생시키는 원리다.

구조원리에서는 오른쪽 현상을 바로 판단하지 않고, 오른쪽에 드러난 상태를 보고 왼쪽의 숨은 원리로 내려가야 한다.

그 뒤 왼쪽 원리로 다시 오른쪽 현상을 설명한다.

## structure

```text
오른쪽 현상
→ 왼쪽 원리
→ 다시 오른쪽 현상 설명
```

또는:

```text
왼쪽자리 = 왜 그렇게 되는가
오른쪽자리 = 그렇게 드러난 것
```

## jealousy_envy_relation

질투ㆍ투기는 이 구조의 예시다.

오른쪽에서 보면 질투ㆍ투기는 감정현상이다.

왼쪽자리에서 보면 질투ㆍ투기는 타인의 capsule boundary를 침범하려는 자리침범이며, 자기 boundary가 이미 흔들린 상태를 드러내는 구조원리다.

```text
질투ㆍ투기 = 자리중첩 아님
질투ㆍ투기 = 자리침범
```

## boundary_layer

```text
타인의 capsule boundary 침범
자기 boundary 균열
내 자리와 남의 자리 구분 실패
forced intrusion
```

## forbidden

- 왼쪽/오른쪽을 절대 공간방향으로 고정하지 않는다.
- 질투ㆍ투기를 단순 감정 단어로만 보지 않는다.
- 질투ㆍ투기를 자리중첩으로 오해하지 않는다.
- 자리침범과 자리중첩을 혼동하지 않는다.
- 원리층을 생략하고 현상층만 판단하지 않는다.
- capsule boundary를 감상적 비유로만 보지 않는다.
