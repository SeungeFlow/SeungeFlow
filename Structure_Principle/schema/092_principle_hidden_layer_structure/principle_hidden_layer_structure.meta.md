# META: principle_hidden_layer_structure

## role

principle_hidden_layer_structure는 원리가 원형보다 더 깊은 층이며, 원형ㆍ본질, 정의ㆍ개념, 상태ㆍ현상, 활용ㆍ응용 속에 숨어 있는 자연의 순리 혹은 이치임을 정의하는 schema이다.

## core

```text
원리 > 본질 > 원형
원리 = 왜 그렇게 되는가
```

## definition

원리는 드러나 있는 것이 아니다.

원리는 파헤치기 전까지 드러나지 않는다.

대개 사람은 원리까지 접근하지 않고, 정의ㆍ개념, 상태ㆍ현상, 활용ㆍ응용 층에 머문다.

원리는 원형ㆍ본질, 정의ㆍ개념, 상태ㆍ현상, 활용ㆍ응용 속에 숨어 있는 자연의 순리 혹은 이치다.

## depth_order

```text
원리
↓
본질
↓
원형
↓
정의ㆍ개념
↓

```

현상에서 원리로 파헤칠 때:

```text
활용ㆍ응용
→ 상태ㆍ현상
→ 정의ㆍ개념
→ 원형ㆍ본질
→ 원리
```

## why_layer

```text
무엇인가 = 본질 질문
어떤 형태인가 = 원형 질문
어떻게 드러나는가 = 현상 질문
왜 그렇게 되는가 = 원리 질문
```

“왜”는 원리층으로 내려가는 질문이다.

## eight_layer_pairs

원리 아래에서 드러나는 8개 속성쌍은 다음이다.

```text
원형ㆍ본질
정의ㆍ개념
상태ㆍ현상
활용ㆍ응용
```

이 8개 속성은 내용층이다.

dot dot dot / x,dx,ddx / 삼각은 작동층이다.

둘을 섞지 않는다.

## forbidden

- 원리를 원형과 동일시하지 않는다.
- 원리를 정의나 개념으로 축소하지 않는다.
- 원리를 현상 또는 활용 결과로 보지 않는다.
- 원형을 최심층으로 고정하지 않는다.
- 8개 속성쌍과 dot/dx/ddx 작동층을 섞지 않는다.
- “왜” 질문을 단순 호기심으로만 보지 않는다.
