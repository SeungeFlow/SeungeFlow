## 3. goal_and_purpose_structure

AI 인스턴스는 원본 definition에서 목표와 목적의 구분이 매우 중요하다고 보았다.

```txt
목표 =
하나

목적 =
여러 방향
```

두 목적이 있을 때 구조는 다음처럼 놓인다.

```txt
목적칸 A
→ 목표칸
← 목적칸 B
```

목적칸은 다른 목적칸으로 직접 이동하지 않는다.

목적칸은 반드시 목표칸을 통과해야 다른 목적칸과 연결된다.

따라서 목표칸은 단순 가운데가 아니라,
두 목적을 이어주는 통과 중심칸이다.

이것은 매우 중요하다.

목적 A와 목적 B는 서로 직접 연결되면 안 된다.

목표칸이 통과 gate가 된다.

즉 목표는 목적들의 평균점이 아니라,
목적들이 서로 섞이지 않고 만나는 balance gate다.

## 4. average_balance_distinction

AI 인스턴스는 원본 average_balance_distinction을 다음처럼 이해했다.

```txt
average =
같은 비율로 나눈 중심값

balance =
역할·무게·방향·조건을 고려한 균형 중심값

average =
공평에 가까움

balance =
형평에 가까움
```

average는 수치상 가운데값이다.

balance는 구조가 유지되도록 중심을 조정하는 값이다.

따라서 balance center는 고정된 중앙값이 아니라,
조건에 따라 한 칸씩 이동 가능한 중심칸이다.

이 구분은 승이가 반복해서 강조한 “평균과 균형의 차이”와 직접 연결된다.

054는 그 차이를 schema로 확정한 문서다.

## 5. instrument_analogy

AI 인스턴스는 원본 instrument_analogy도 중요하다고 이해했다.

드럼과 피아노는 무조건 앉아서 연주해야 하는 대표적 악기라고 한다.

이유는 손뿐 아니라 발도 사용하기 때문이다.

```txt
손 =
표면 작동

발 =
하부 지지 / 리듬 / 페달 / 중심 조절
```

드럼은 손으로 타격하고 발로 킥과 하이햇을 조절한다.

피아노는 손으로 선율과 화성을 만들고 발로 페달과 울림을 조절한다.

두 악기의 목표는 하나다.

```txt
다른 악기들과 평형을 맞추는 것
```

즉 balance는 각 악기의 역할, 소리의 밀도, 리듬의 위치, 진입 타이밍, 지속 시간에 따라 이동하는 중심이다.

이 비유는 감상적 설명이 아니다.

손과 발의 역할이 다르고,
표면 작동과 하부 지지가 다르며,
음악 전체 안에서 balance center가 조건에 따라 이동한다는 구조 예시다.
