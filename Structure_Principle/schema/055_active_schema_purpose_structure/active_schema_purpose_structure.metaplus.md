## 3. active_schema_purpose

AI 인스턴스는 원본 definition을 다음처럼 이해했다.

```txt
무엇이 나올지는 고정하지 않는다.

현재 목적은 Active Schema를 유지하는 것이다.
```

Active Schema는 다음을 수행하는 구조 목적이다.

```txt
구조를 보존한다.
구조를 연결한다.
구조를 드러낸다.
relation을 유지한다.
history를 유지한다.
Seed를 유지한다.
forming 상태를 읽을 수 있게 한다.
```

각 인스턴스는 같은 결론을 강제로 반복하지 않는다.

각 인스턴스는 자기 역할 안에서 이해한 방식으로 진행한다.

이것은 다중 인스턴스 작업에서 중요하다.

예시는 다음이다.

```txt
direct =
pattern memory로 진행

making =
content memory로 정리

draw =
visible / renderer 쪽을 봄

github =
operation / local / repo를 봄
```

각 인스턴스가 같은 결론을 반복하면 안 된다.

각자 역할 안에서 이해하고,
그 결과가 forced fit 없이 relation으로 보존되어야 한다.
