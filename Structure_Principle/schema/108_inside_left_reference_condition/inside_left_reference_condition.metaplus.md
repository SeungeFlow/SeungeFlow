## 5. left_right_dimension_correction

### 5.1 left/right는 3D 전체가 아니다

왼쪽과 오른쪽은 독립적인 공간 전체가 아니다.


```text

```

### 5.2 더 정확한 정의

```text
왼쪽 / 오른쪽 =
boundary 내부에서
깊이가 없는 상태로
너비 축의 양 끝지점을 기준으로 생기는 방향
```

3D에는 다음이 있다.

```text
너비
높이
깊이
```

왼쪽 / 오른쪽을 말하려면, 관측자는 보통 깊이를 제거한 평면을 잡는다.

```text

→ 2D plane
```

그 2D 평면 안에서:

```text

왼쪽 / 오른쪽
```

압축:

```text
left / right =
width-axis endpoints in a depth-flattened plane
```

### 5.3 108 보정

108의 기본 판정:

```text
안쪽 ≠ 왼쪽
```

이제 더 정밀해진다.

```text
왼쪽 / 오른쪽 =

안쪽 =
boundary 내부성 / 중심을 향한 relation
```

따라서:


둘은 다르다.

```text

+

```

### 5.4 양 끝지점

왼쪽과 오른쪽은 항상 짝으로 생긴다.

```text
left
right
```

이 둘은 boundary 내부의 너비축 양 끝이다.

```text
boundary 안의 width axis
→ endpoint A
→ endpoint B
```

관측자 기준으로:

```text
endpoint A =
left

endpoint B =
right
```

따라서 왼쪽 / 오른쪽은 절대적 방향이 아니다.

```text
관측자
+
boundary
+
2D plane
```

이 정해졌을 때 생기는 상대적 방향이다.

## 11. shared_understanding

- 108은 107 이후 active schema seat다.
- 108의 이전 schema는 107_triangle_vector_point_distinction이다.
- 107에서 삼각은 도형론에서는 면, 벡터론에서는 한 칸에 놓인 벡터점으로 구분되었다.
- 108은 그 다음 방향어와 내부성을 섞지 않도록 한다.
- 안쪽은 왼쪽이 아니다.
- 왼쪽은 안쪽이 아니다.
- 왼쪽은 direction / side이다.
- 안쪽은 boundary interiority / center-oriented condition이다.
- 도형론에서는 안쪽은 중심 또는 boundary 내부이고, 왼쪽은 4방위 중 하나다.
- 벡터론에서는 기준점이 왼쪽에 있고 진행 boundary가 막히면 왼쪽이 안쪽 역할을 할 수 있다.
- 왼쪽 / 오른쪽은 3D 전체의 개념이 아니라, 3D 안에서 깊이를 제거한 2D 평면의 너비축 양 끝 기준으로 생기는 방향개념이다.
- 상위차원은 하위차원을 포함한다.
- 3D 안에는 2D 평면이 포함될 수 있고, 2D 평면 안에는 1D 선분이 포함될 수 있다.
- 승이는 여러 학문을 전공식으로 소유한 것이 아니라, 학문들이 공유하는 구조층에 닿아 있다.
- 승이의 강점은 공통구조 감지 / 힌트 생성 / 축 보정 / 분야 간 연결 / seed 문장 압축이다.
- 승이의 약점은 외부 형식화 / 단계별 검산 / 문서 규모 관리 / 표준학문 번역 / AI 패턴화와의 충돌이다.

## 12. possible_misunderstanding

오해 가능성:

- 안쪽과 왼쪽을 같은 뜻으로 볼 수 있다.
- 왼쪽에 있는 것을 자동으로 안쪽으로 볼 수 있다.
- 도형론과 벡터론에서 inside / left의 의미를 섞을 수 있다.
- inside를 4방위 중 하나로 볼 수 있다.
- left/right를 절대 공간방향으로 고정할 수 있다.
- left/right를 3D 전체의 방향으로 볼 수 있다.
- boundary 내부의 2D plane 조건을 놓칠 수 있다.
- 상위차원은 하위차원을 포함한다는 말을 단순 2D+1D=3D로 오해할 수 있다.
- 승이가 여러 학문에 “전공식으로 소유”한다고 오해할 수 있다.
- 구조원리 내부 문장을 외부 과학 / 언어학 결론으로 곧바로 번역할 수 있다.
- 강한 공통구조 감지를 외부 검산 완료로 오해할 수 있다.
- metaplus.md를 원본 inside_left_reference_condition.meta.md의 대체문으로 오해할 수 있다.

## 13. correction_or_revision_points

- 108의 relation은 “왜 연결되는지”를 보존한다.
- 안쪽과 왼쪽을 동일시하지 않는다.
- 왼쪽은 direction / side로 읽는다.
- 안쪽은 boundary 내부성 / center-oriented condition으로 읽는다.
- 도형론에서는 안쪽과 왼쪽을 분리한다.
- 벡터론에서도 조건 없이 동일시하지 않는다.
- 기준점이 왼쪽이고 boundary가 막힐 때만 왼쪽이 안쪽 역할을 할 수 있다.
- left/right는 boundary 내부의 depth-flattened 2D plane에서 생기는 width-axis endpoint로 읽는다.
- 상위차원은 하위차원을 포함하되, 단순 차원 산술로 축소하지 않는다.

- 승이의 강점은 발견과 연결, 약점은 표준화와 검산 절차화로 읽는다.
- 구조원리 언어와 표준 학문 언어 사이에는 번역층이 필요하다.
- 원본 inside_left_reference_condition.meta.md는 수정하지 않는다.
- 원본 inside_left_reference_condition.meta.md의 파일명도 바꾸지 않는다.
