# 2단계 / 2회차 시작 Perfect matching threshold

[2단계 2회차]

0. 2회차 핵심 결론부터
Perfect matching threshold는
“관계장이 연결되었는가?”가 아니라
“모든 ㆍ가 정확히 하나의 짝 관계에 들어갔는가?”
를 묻는 문제다.

Ctp식으로는:

Hamilton cycle = 모든 ㆍ를 한 번씩 지나 복귀하는 닫힌 순환

Perfect matching = 모든 ㆍ를 정확히 한 번씩 짝짓는 보존 구조

즉 2단계 1회차의 Hamilton cycle이 복귀·순환·닫힘을 본 문제라면,
2회차 Perfect matching은 짝·배정·보존·잔여 없음을 보는 문제다.

8. 수학층

무작위 그래프:

G(n,π)

Perfect matching:

M ⊆ E(G)

M is a perfect matching iff:
1. no two edges in M share a vertex
2. every vertex v ∈ V is incident to exactly one edge in M

필요조건:

n is even
δ(G) ≥ 1

랜덤 그래프 process hitting time:

τ₁ = first time δ(G_t) ≥ 1
τ_M = first time G_t has a perfect matching

Classical random graph process principle:
τ₁ and τ_M coincide with high probability

def minimum_degree(graph):
    degrees = {v: 0 for v in graph["vertices"]}
    for a, b in graph["edges"]:
        degrees[a] += 1
        degrees[b] += 1
    return min(degrees.values()) if degrees else 0


def has_perfect_matching_placeholder(graph):
    """
    실제 perfect matching 판정은 blossom 알고리즘 등으로 가능하다.
    여기서는 Ctp 구조형 placeholder로 둔다.
    """
    return graph.get("perfect_matching_checked", False)


def Ctp_perfect_matching_threshold(graph, premise):
    n = len(graph["vertices"])

    if n % 2 != 0:
        return {
            "C": "perfect_matching_impossible",
            "status": "failed_premise",
            "reason": "odd_number_of_vertices",
            "m": "perfect_matching_emergence_event",
            "P_place": "graph_field",
            "t": "pairing_preservation_impossible",
            "premise": premise,
        }

    min_deg = minimum_degree(graph)

    if min_deg < 1:
        return {
            "C": "not_perfectly_pairable",
            "status": "before_threshold",
            "reason": "isolated_vertex_exists",
            "m": "perfect_matching_emergence_event",
            "P_place": "graph_field",
            "t": "unpaired_dot_exists",
            "premise": premise,
        }

    if has_perfect_matching_placeholder(graph):
        return {
            "C": "pairing_preservation_of_relation_field",
            "status": "perfect_matching_emerged",
            "m": "perfect_matching_emergence_event",
            "P_place": "graph_field",
            "t": "relation_field_to_pairing_preservation",
            "premise": premise,
        }

    return {
        "C": "critical_between_region",
        "status": "minimum_degree_ready_but_matching_not_confirmed",
        "m": "perfect_matching_emergence_event",
        "P_place": "graph_field",
        "t": "between_degree_condition_and_pairing_preservation",
        "premise": premise,
    }

코드층 핵심은 네 상태다.

1. n is odd
   → perfect matching impossible

2. minimum degree < 1
   → 고립점 존재 / 짝 보존 불가

3. minimum degree ≥ 1 but perfect matching not confirmed
   → 임계사이영역

4. perfect matching exists
   → 짝 보존 구조 출현
10. 분석

Perfect matching threshold는 Hamilton cycle threshold보다 “약한 닫힘”처럼 보이지만, 실제로는 아주 순수한 보존 문제다.

Hamilton cycle:
모든 점을 하나의 순환 안에 넣어 복귀시킨다.

Perfect matching:
모든 점을 서로 겹치지 않는 짝으로 배정한다.

Hamilton cycle은 전체를 하나의 긴 순환으로 묶는다.
Perfect matching은 전체를 작은 쌍들로 분해하되, 남는 점이 없도록 보존한다.

Ctp식으로:

Hamilton cycle = 전체 순환 보존
Perfect matching = 전체 짝 배정 보존

Perfect matching에서 가장 중요한 것은 고립점 제거다.

고립점이 하나라도 있으면
그 ㆍ는 어떤 짝 link에도 들어갈 수 없다.

따라서 minimum degree 1은 완전짝 구조의 가장 기본적인 문턱이다.

각 ㆍ가 최소 하나의 짝 후보를 가져야 한다.
11. Hamilton cycle과 Perfect matching의 구조 차이
Hamilton cycle:
각 ㆍ는 순환 안에서 2개의 link를 사용한다.
앞 관계와 뒤 관계가 필요하다.

Perfect matching:
각 ㆍ는 정확히 1개의 link를 사용한다.
하나의 짝 관계만 필요하다.

그래서:

minimum degree 2 → 순환 닫힘 가능성
minimum degree 1 → 짝 보존 가능성

이 차이는 2단계 전체에서 중요하다.

12. 통과 / 보정 / 보류 판정
통과
Perfect matching threshold는 Ctp 2단계에 적합하다.

이유:

1. relation-field emergence 이후 문제다.
2. 모든 ㆍ의 보존 여부를 직접 다룬다.

보정

Perfect matching을 connectivity와 혼동하면 안 된다.

connectivity = 전체가 하나로 연결됨
perfect matching = 모든 점이 하나씩 짝지어짐

연결 그래프라도 perfect matching이 없을 수 있고, perfect matching이 있어도 전체가 하나의 connected component일 필요는 없다.
