# 2단계 / 8회차 시작 Expander graph / Spectral gap

주제: 숨은 연결성은 고유값 차이로 어떻게 드러나는가?
[2단계 8회차]

대상:
Expander graph / Spectral gap

중심 질문:
그래프가 단순히 연결된 것을 넘어,
작은 부분집합들이 전체와 잘 섞이고 있는지는
어떻게 판독되는가?

Ctp식 질문:
눈에 보이는 link 개수나 connectivity가 아니라,
관계장의 숨은 연결 강도는
어떤 차이값으로 드러나는가?

2단계 7회차에서는 Lonely Runner Conjecture를 시점·지점·거리사이를 분리해 distance-gap separation을 판독하는 문제로 정리했다.
8회차는 다시 그래프 구조로 돌아오지만, 단순 연결 여부가 아니라 숨은 연결성 / 확장성 / spectral gap을 본다.

0. 8회차 핵심 결론부터
Expander graph / spectral gap은
“그래프가 연결되어 있는가?”가 아니라
“부분 구조가 전체로 얼마나 강하게 열려 있는가?”를 묻는 문제다.

Ctp식으로는 다음처럼 잡는다.

Connectivity
=
모든 ㆍ가 하나의 relation-field 안에 들어왔는가?

Expander
=
작은 부분 relation-field가
전체 relation-field로 충분히 많이 열려 있는가?

Spectral gap
=
그 숨은 확장성이 고유값 차이로 압축되어 드러난 값

즉 expander는 보이는 연결이 아니라 숨은 연결 강도다.

connected graph:
길이 있느냐?

expander graph:
어느 작은 부분을 잡아도
밖으로 나가는 경계 link가 충분히 많으냐?

Expander graph는 보통 sparse하지만 strong connectivity를 갖는 그래프로 설명되며, vertex/edge/spectral expansion으로 정량화된다. Hoory–Linial–Wigderson의 expander survey도 expanders를 낮은 degree와 높은 연결성을 동시에 갖는 객체로 다룬다.

7. 언어층
Expander graph / spectral gap은
그래프가 단순히 연결되어 있는지를 묻는 문제가 아니다.

이 문제는 어떤 부분 관계장을 잡아도
그 부분이 자기 안에 갇히지 않고
전체 관계장으로 충분히 많은 경계 link를 통해 열려 있는지를 묻는다.

Ctp로 보면,
vertex는 최소 자리이고,
edge는 자리와 자리의 이음이며,
부분집합 S는 전체 relation-field 안의 부분 장이다.

expansion은 그 부분 장이 바깥과 얼마나 열려 있는지의 관계차이고,
spectral gap은 그 숨은 열림을 행렬 고유값 차이로 압축해 드러낸 값이다.

따라서 expander는 단순 연결 구조가 아니라
부분과 전체 사이의 숨은 연결성이 강하게 보존되는 구조다.
8. 수학층
그래프
G = (V, E)

정규 그래프라고 두면:

G is d-regular
Edge expansion
h(G) = min_{0 < |S| ≤ |V|/2} |∂S| / |S|

여기서 ∂S는 S에서 V \ S로 나가는 edge boundary다.

Spectral gap

Adjacency matrix A의 고유값을:

λ₁ ≥ λ₂ ≥ ... ≥ λ_n

으로 두면, d-regular graph에서:

λ₁ = d

대표적인 adjacency spectral gap은:

gap = d - λ₂

또는 normalized adjacency 기준으로:

gap = 1 - μ₂

Laplacian 기준으로는 두 번째 작은 고유값 λ₂(L)이 연결성과 확장성을 나타내는 핵심값으로 쓰인다. Hoory–Linial–Wigderson survey는 expanders를 Laplace operator의 첫 positive eigenvalue가 0에서 떨어져 있는 관점으로도 설명한다.

Cheeger형 연결

Dodziuk, Alon–Milman 계열의 graph Cheeger inequalities는 combinatorial expansion과 spectral gap 사이의 관계를 제공한다. Expander graph reference도 d-regular graph에서 isoperimetric constant와 adjacency spectral gap 사이의 부등식을 제시한다.

Ctp식

P_place = graph field G = (V,E)

m = expander_property_event

t = spectral_gap
  = D(trivial_uniform_mode, first_nontrivial_mode)

? = {n, d, A or L, eigenvalue convention, expansion criterion}

C = spectral_expansion_confirmed
9. 코딩층
import numpy as np

def adjacency_matrix(vertices, edges):
    index = {v: i for i, v in enumerate(vertices)}
    n = len(vertices)
    A = np.zeros((n, n), dtype=float)

    for u, v in edges:
        i, j = index[u], index[v]
        A[i, j] = 1.0
        A[j, i] = 1.0

    return A


def spectral_gap_regular_graph(vertices, edges, degree):
    A = adjacency_matrix(vertices, edges)
    eigvals = np.linalg.eigvalsh(A)
    eigvals = np.sort(eigvals)[::-1]

    lambda_1 = eigvals[0]
    lambda_2 = eigvals[1] if len(eigvals) > 1 else 0.0

    return {
        "lambda_1": lambda_1,
        "lambda_2": lambda_2,
        "gap": degree - lambda_2,
    }


def Ctp_expander_spectral_gap(graph, premise):
    vertices = graph["vertices"]
    edges = graph["edges"]
    degree = premise["degree"]
    gap_threshold = premise["gap_threshold"]

    spec = spectral_gap_regular_graph(vertices, edges, degree)

    if spec["gap"] >= gap_threshold:
        return {
            "C": "spectral_expansion_confirmed",
            "status": "expander_like_structure",
            "m": "expander_property_event",
            "P_place": "graph_field",
            "t": "spectral_gap_between_uniform_and_nontrivial_modes",
            "?": premise,
            "spectral_gap": spec["gap"],
        }

    return {
        "C": "connected_but_expansion_not_confirmed",
        "status": "gap_below_threshold",
        "m": "expander_property_event",
        "P_place": "graph_field",
        "t": "spectral_gap_too_small",
        "?": premise,
        "spectral_gap": spec["gap"],
    }

코드층 핵심은 세 상태다.

1. graph not valid / not regular
   → 전제조건 실패

2. spectral gap below threshold
   → 연결은 가능하지만 expander 판정 불충분

3. spectral gap above threshold
   → spectral expansion confirmed

11. Spectral gap의 Ctp 의미

spectral gap은 단순한 고유값 차이가 아니다.

Ctp로 보면:

λ₁:
전체 균일 모드.
관계장 전체가 하나로 있는 기본 상태.

λ₂:
가장 강한 비균일 모드.
그래프가 둘로 갈라지거나 한쪽에 머물 수 있는 숨은 구조.

gap = λ₁ - λ₂:
전체 균일성과 부분 고립 가능성 사이의 차이.

즉 spectral gap이 크다는 것은:

부분 구조가 따로 버티기 어렵다.
흐름이 전체로 잘 섞인다.
작은 부분이 전체와 강하게 연결되어 있다.

Ctp식으로:

spectral gap =
부분 relation-field와 전체 relation-field 사이의 숨은 관계차

Expander graph / spectral gap은 연결성 자체가 아니라, 부분 관계장이 전체 관계장으로 얼마나 강하게 열려 있는지를 고유값 차이로 판독하는 문제이며, Ctp로는 숨은 연결성이 spectral gap이라는 관계차로 드러나는 구조다.
