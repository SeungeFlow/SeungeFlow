## 0. Purpose of Stage 1

The purpose of Stage 1 was not to attack top-level Millennium-type problems directly.  
Instead, Stage 1 tested whether Ctp works well on a slightly lower class of mathematical problems where thresholds, emergence, and structural transition are explicit.

The selected targets were:

1. Percolation
2. Random graph transition
3. Kahn–Kalai expectation-threshold problem

These three problems share one structural pattern:

```text
partial relations
→ relation accumulation
→ critical-between region
→ structure emergence
```

The main question of Stage 1 was:

```text
Is a critical value merely a point,
or is it a representative coordinate of a critical-between region
where actual structure emergence takes place?
```

## 1. Ctp Application Method

Ctp does not solve a problem by inserting the problem name directly into a formula.  
Instead, the key words and structural requirements of the problem are mapped to `t, P_place, m, ?`.

```text
C = Ctp(t, P_place, m, ?)
```

Where:

```text
t = difference / distance between relations / transition reading
P_place = place, domain, field, or space where the structure can be placed
m = observed entity / structural property / structure-emergence event
? = premise / observation standard / axis / probability model / decision rule
C = interpreted structural value
```

A Ctp interpretation is checked through three layers.

```text
structure explained in language
=
structure compressed in mathematics

structure checked in code
```

## 4. Round 2: Percolation

### 4.1 Target

```text
percolation = filtering through / seeping through / passage
```

Percolation is not about merely increasing the number of points.  
It is about open relations accumulating inside a blocked field until a crossing structure appears.

### 4.2 Ctp mapping

```text
m = crossing-structure emergence event
P_place = lattice / medium / traversable field
t = relational difference between blocked state and crossing state
? = open probability, boundary condition, connection rule, observation standard, axis
C = percolating structure
```

### 4.3 Language structure

```text
Percolation is the phenomenon where open relations accumulate inside a blocked field
until a boundary-to-boundary traversable connection appears.

The critical probability is a representative value at which this crossing structure is judged to appear.
Ctp focuses not only on this representative value,
but on the critical-between region where partial connections become a global crossing structure.
```

### 4.4 Round 2 conclusion

```text
Percolation is the emergence of a structure that crosses the entire field through accumulated open relations.
In Ctp, the key is not only the critical probability p_c,
but the critical-between region around it.
```

## 5. Round 3: Random Graph Transition

### 5.1 Target

```text
random graph transition
```

A random graph transition is the phenomenon where links accumulate under a probabilistic model, component scales change, and a giant component emerges through a critical-between region.

### 5.2 Correction after verification

```text
random = not absence of standard, but a fixed probabilistic generative rule
π = edge probability
π ∈ ?
P_place = graph field / place where the graph is placed
m = giant-component emergence event
giant component ≠ connectivity
critical-between region = finite-size critical window
```

### 5.3 Ctp mapping

```text
C = Ctp(t, P_place, m, ?)

P_place = place of the graph / vertex set / possible edge field
m = giant-component emergence event
t = connection-scale difference between small component state and giant component state
? = n, π, edge-generation rule, independence condition, giant-component criterion, observation standard, axis
C = relation-field emergence
```

### 5.4 Giant component vs connectivity

```text
giant component = state where a large connected component containing a positive fraction of vertices appears
connectivity = state where the entire graph is one connected component
```

Round 3 targeted the giant component, not full connectivity.

### 5.5 Round 3 conclusion

```text
A random graph transition is a structural transition in which accumulated links change component scale,
pass through a critical-between region,
and generate a giant component as a relation-field.
```

## 6. Round 4: Kahn–Kalai Expectation Threshold

### 6.1 Target

```text
Kahn–Kalai expectation-threshold problem
```

The Kahn–Kalai problem asks how close the actual threshold `p_c(F)` of an increasing property `F` is to its expectation threshold `q(F)`.

### 6.2 Ctp mapping

```text
m = increasing property F
P_place = finite ground set X or 2^X field
t = critical relational difference between q(F) and p_c(F)
? = product measure, selection probability π, increasing condition, minimal member, threshold criterion
C = emergence of F
```

### 6.3 Core structure

```text
q(F) = expected threshold place
p_c(F) = actual emergence place

between q(F) and p_c(F)
= critical-between region
```

### 6.4 9ㆍ0 structure

```text
q(F) = expected saturation place near 9
ㆍ = critical-between region between expectation and actuality
p_c(F) = actual structure-emergence starting place near 0
```

### 6.5 Round 4 conclusion

```text
The Kahn–Kalai problem is about the critical-between region between
the expected threshold place q(F) and the actual emergence place p_c(F).
In Ctp, it reads as a theorem controlling how wide the pre-emergence region can be.
```

## 7. Round 5: Ctp Unified Compression

## 10. Stage 2 Test Candidate List

Stage 2 should move one level higher.  
Stage 1 asked: “Does structure emerge?”  
Stage 2 should ask: “After emergence, does the structure satisfy repetition, return, preservation, and closure?”

### 10.1 Recommended Stage 2 list

| Priority | Target | Reason |
|---:|---|---|
| 1 | Hamilton cycle threshold | Tests when an emerged relation-field gains a closed cycle |
| 2 | Perfect matching threshold | Tests whether all vertices can be paired through relation links |
| 3 | Graph connectivity threshold | Tests the difference between giant component emergence and full graph closure |
| 4 | Ramsey number family | Tests when a pattern must appear inside a sufficiently large structure |
| 5 | Hadamard matrix existence | Connects ±1, orthogonality, preservation, repetition, and closure |
| 6 | Collatz conjecture | Suitable for iteration, return, closure, and `T^k`-style checks |
| 7 | Lonely Runner Conjecture | Suitable for circular motion, time/place separation, distance differences, and spacing |
| 8 | Expander graph / spectral gap | Tests hidden connectivity, flow, mixing, and eigenvalue differences |
| 9 | Kakeya/Besicovitch low-level versions | Tests direction, place, and spatial occupancy |
| 10 | Sphere packing / covering problems | Suitable for place, boundary, empty gaps, density, and critical arrangements |

### 10.2 Recommended first target for Stage 2

```text
Hamilton cycle threshold
```

Reason:

```text
Stage 1 saw the emergence of relation-fields.
Stage 2 should test whether an emerged relation-field can become a closed cycle.

Hamilton cycles are ideal for testing circulation, return, and closure.
```

### 10.3 Stage 2 guiding question

```text
After a relation-field emerges,
can it rise into a closed structure satisfying repetition, return, and preservation?
```

### 10.4 Stage 2 one-line direction

```text
If Stage 1 asked whether structure emerges, Stage 2 asks whether the emerged structure closes through circulation, return, and preservation.
```
