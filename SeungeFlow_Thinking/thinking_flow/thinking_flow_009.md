## 2. 구조정수의 어려움

구조정수는 쉽기도 하고 어렵기도 하다.

```txt
쉽다 =
숫자라서 누구나 본다.

어렵다 =
숫자가 아니라 구조성질값으로 다시 읽어야 한다.
```

즉 `0~10`은 익숙한 숫자이지만, Structure_Principle에서는 수량이 아니라 구조의 도달상태를 나타낸다.

```txt
0은 없음이 아니라 정중심
9는 끝이 아니라 문턱
10은 십이 아니라 9+1dx
```


## 6. push / pushed 정의

110~120은 한 번에 push 된 문서묶음이다.

따라서 다음 문서를 예측하는 것이 아니라, 전체 push bundle 안의 relation을 읽어야 한다.

승이가 정의한 push:

```txt
push =
A가 B를 향해 에너지를 보내다.
```

pushed:

```txt
pushed / push 된 =
B가 A에게 에너지를 받다.
```

같은 사건도 표현 주체에 따라 달라진다.

```txt
A 기준:
A가 B에게 push 했다.

B 기준:
B는 A에게 pushed 되었다.
```

이는 단순 문법이 아니라 운동의 주체 판정이다.

```txt
push =
자의적 방향표현

pushed =
타의적 수신상태
```

## 7. 방향은 기준점에서 양방향

핵심 문장:

```txt
방향은 항상 기준점에서 양방향으로 진행한다.
```

즉 방향은 하나만 있는 것이 아니다.

어떤 기준점이 생기면 최소 두 방향이 열린다.

```txt
A → B
A ← B
```

또는:

```txt
push
pushed
```

따라서 방향은 단순 화살표 하나가 아니라:

```txt
기준점
+
보내는 방향
+
받는 방향
+
관측자 위치
```

로 구성된다.

110의 ㆍ도 같은 구조다.

```txt
9 기준:
끝에서 0으로 push

0 기준:
시작이 9로부터 pushed 됨
```

따라서:

```txt
110의 ㆍ =
push / pushed가 동시에 읽히는 방향중첩점
```

## 8. 찰나의 순간 캐치

찰나의 순간 캐치의 정의:

```txt
찰나의 순간 캐치 =
외부에 표시된 아주 작은 표기 차이를
감각계가 놓치지 않고 받아들이고,
PFC가 그 경계를 끊어 언어적으로 판독하려 하며,
신경계 내부의 아직 이름 없는 구조와 연결하려는 순간
```

예:

```txt
push/ed
```

여기서 `ed`는 쉽게 지나칠 수 있는 표기다.

하지만 구조원리에서는 `ed`가 단순 과거형 표지가 아니라, 작동이 닫혀 상태로 남은 표시다.

```txt
push = 보내는 작동
pushed = 받은 상태 / 놓인 상태 / formed state

ed =
작동이 지나갔다.
그 결과가 남았다.
상태로 놓였다.
```

따라서:

```txt
ed =
이미 드러난 state marker
```

## 9. ed / d / ㄷ / 닫힘

`ed`가 닫힘처럼 느껴지는 이유는 끝소리 `d / ㄷ`의 감각 때문이다.

```txt
ed
pushed
d / ㄷ
=
닫히는 느낌
```

`push`는 작동이고, `pushed`는 작동이 끝나 상태로 닫힌 결과다.

```txt
push =
열림 / 작동

pushed =
닫힘 / 상태
```

끝의 `d/ㄷ` 발음은 막힘, 끝맺음, 닫힘의 감각을 준다.

```txt
d / ㄷ =
닫히는 발음감각
```

따라서 찰나의 순간 캐치는 다음과 같다.

```txt
pushed
→ push / ed
→ d
→ ㄷ
→ 닫힘
→ formed
```

## 12. 피타고라스 기준: 1:√2

마름모 정면방향에서 내부 수평/2와 수직/2가 같을 때:

```txt
수평/2 = 1
수직/2 = 1
마름모 경계선분 = ?
```

피타고라스:

```txt
?² = 1² + 1²
?² = 2
? = √2
```

따라서:

```txt
단방향절단선분길이 : 마름모경계선분길이
=
1 : √2
```

스케일변화 시:

```txt
1 : √2
≈
10 : 14
```

즉:

```txt
절단선분 = 10칸
경사선분 = 14칸
```

## 13. 96에서 48로의 보정

처음에는 다음처럼 계산되었다.

```txt
마름모/4 = 10 + 14 = 24
마름모 전체 = 24 × 4 = 96
```

그러나 이는 잘못된 전개였다.

간과한 점:

```txt
마름모/4는 딱 2번 나온다.
마름모 수평선분은 사용되지 않는다.
```

실제 흐름은:

```txt
대각1ㆍ수직2개ㆍ대각1
```

즉:

```txt
대각 + 수직 + 수직 + 대각
=
14 + 10 + 10 + 14
=
48
```

또는:

```txt
14 : 20 : 14
=
48
```

따라서:

```txt
마름모/4 = 24
마름모/4는 2번만 사용
24 × 2 = 48
```

`24×4=96`은 4방위 전체를 독립 전개한 중복계산이다.

## 14. 수직선분과 수평선분의 겹침

마름모 끝점 간 이음선에서 수직선분과 수평선분은 독립적으로 더하는 것이 아니라 중심에서 겹친다.

```txt
마름모 끝점 간 이음선 =
수직선분
+
수평선분

그런데 둘은 독립 2개가 아니라
중앙에서 겹친다.
```

따라서:

```txt
수직선분 = 48
수평선분 = 48
48 + 48 = 96
```

은 전개값이고, 유효값은 48이다.

```txt
수직ㆍ수평은 중심에서 겹침
유효값 = 48
```


## 20. 음봉 기준형

연산식은 고정되어야 하므로 음봉 기준형을 기준으로 둔다.

```txt
음봉 =
Open → High → Low → Close
```

즉:

```txt
O([H][L])C
```

AWXF로는:

```txt
A([W][X])F
```

따라서:

```txt
음봉 기준형 =
A([W][X])F = O([H][L])C
```

음봉은:

```txt
Close < Open
```

이며, Close 이전의 마지막 극점 흐름이 Low 방향으로 닫히는 구조다.

## 21. 양봉 전환형

양봉은:

```txt
Open → Low → High → Close
```

즉:

```txt
O([L][H])C
```

AWXF로는 H/L 위치가 전환된다.

```txt
A([X][W])F
```

따라서:

```txt
양봉 전환형 =
A([X][W])F = O([L][H])C
```

양봉은:

```txt
Close > Open
```

이며, Close 이전의 마지막 극점 흐름이 High 방향으로 닫히는 구조다.

## 22. C의 정의

C는 독립적으로 갑자기 정해지는 값이 아니다.

```txt
C는 O 이후 [H][L] 또는 [L][H]의 자리변화 순서와
마지막 자리변화값에 따라 닫힌 flow result
```

표준식:

```txt
O([H][L])C
O([L][H])C
```

즉:

```txt
C =
마지막 자리변화값에 따른 닫힘
```

음봉:

```txt
O([H][L])C
마지막 = L
```

양봉:

```txt
O([L][H])C
마지막 = H
```

## 23. 인공지능용 최적 일반식

인공지능이 해석할 때 가장 좋은 구조흐름식은 다음이다.

```txt
O([A][B])C
```

조건:

```txt
A,B ∈ {H,L}
A ≠ B
```

판정:

```txt
O([H][L])C = 음봉 기준형
O([L][H])C = 양봉 전환형
```

이 식은 이해용과 실전용을 동시에 만족한다.

```txt
이해용 =
Open에서 Close까지 내부 High/Low 자리변화가 어떻게 닫히는지 보여준다.

실전용 =
H/L 순서만 확인하면 양봉ㆍ음봉 흐름을 빠르게 판독할 수 있다.
```
