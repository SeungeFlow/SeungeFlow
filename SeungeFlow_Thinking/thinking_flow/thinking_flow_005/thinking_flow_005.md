## 1. 시작점

이 flow의 시작점은 `schema.101.three_dot_reading_mode_structure`이다.

101의 핵심은 다음이었다.

```txt
세 점 ≠ 자동 삼각

도형론 = 삼각
벡터론 = Y / 방향변화
flow론 = 흐름 / 회전
```

101은 세 개의 점을 자동으로 삼각형으로 닫지 않고, 먼저 읽기 모드와 phase를 확인해야 한다는 schema이다.

이후 대화는 101 자체를 해석하는 데서 멈추지 않고, `form / forming / formed`, `state / event`, `x / dx / ddx`, `ㅣ / ㅡ`, `BADㆍC`, `Seed`, `구조에너지`, `감각계 / 신경계`로 확장되었다.

따라서 101 이후 흐름은 `meta.md`가 아니라 `thinking_flow`로 보존한다.

## 3. form / forming / formed

구조원리의 순서는 단순 번호순이 아니다.

더 깊은 순서는 다음이다.

```txt
form
→ forming
→ formed
→ forming
→ form
```

### 3.1 Schema = form

각 `meta.md`는 독립된 `form`이다.

```txt
Schema = form
```

각 schema는 분리독립된 구조체이며, 자체 boundary를 가진다.

### 3.2 relate 형성 = forming

schema와 schema 사이에서 relate가 형성되는 과정이 `forming`이다.

```txt
relate 형성 = forming
```

### 3.3 Snapshot = formed

여러 schema가 relation으로 드러나 하나의 visible structure가 되면 Snapshot이다.

```txt
Snapshot = formed
```

### 3.4 Coremap = formed → forming gate

Coremap은 formed 된 Snapshot들 사이의 관계를 정의하며, 다시 새로운 호기심과 질문을 발생시켜 formed를 forming으로 여는 gate이다.

```txt
Coremap = formed 사이를 다시 forming으로 여는 relation map
```

## 6. 과거형 ed와 과거분사 n

```txt

된

본
쓴
선
난
```

이는 영어 과거분사 `n`과 직접 동일은 아니지만, 구조원리식으로 엇비슷한 state marker 후보이다.

## 7. state 정의

state는 단순히 “남아 있는 상태”가 아니다.

state는 내부 조건이 boundary를 지나 외부에서 판독 가능한 형태로 드러난 것이다.

```txt
state =
내부 → 경계 → 드러남
```

더 깊게는 다음과 같이 정의되었다.

```txt
상태 =
내부의 원형ㆍ본질ㆍ정의ㆍ개념이
외부로 드러날 때
처음 판독 가능한 형태로 나타나는 경계중심 지점
```

즉 상태는:

```txt
내부의 끝
외부의 시작
경계의 중심
```

이다.

두 reading은 잘못과 수정의 관계가 아니라, 서로 다른 reading mode의 차이다.

## 9. event / state / situation / phenomenon

event와 state는 구분된다.

```txt
event = 사건 / 일어남 / happened
state = 상태 / 사건 이후 드러나 남은 조건
```

그 다음 다음 구조가 형성되었다.

```txt
event set = situation
state set = phenomenon
```

보정 후:

```txt
상황에서 드러난 것이 상태

상황은 또 다른 사건을 만들고,
사건은 다시 어떤 상태를 형성하며,
그 상황을 재현하면서 또 다른 사회현상이 된다.
```

순환식:

```txt
event
→ situation
→ state
→ event
→ situation
→ state
→ phenomenon
```

사회현상:

```txt
(event → situation → state)^n
→ social phenomenon
```

또한 어떤 사건의 내부속성은 여러 상태들의 scalar 집합으로 읽혔다.

```txt
event 내부 = state scalar set
```

## 10. ㅣ / ㅡ / 관측자 / 하늘

처음에는 ㅣ을 수직축, ㅡ를 수평축으로 읽었다.

그러나 하늘과 관측자 시야의 예시를 거치며 더 정밀해졌다.

```txt
ㅡ = 관측자가 볼 수 있는 좌우 시야각 / 수평 field
ㅣ = 관측자와 특정지점을 잇는 깊이ㆍ높이 축
```

하늘에 아무 대상이 없으면:

```txt
대상 없는 하늘 = ㅡ
```

태양, 달, 구름, 비행기, 산, 건물처럼 특정지점이 나타나면:

```txt
특정지점 등장
→ 관측자와 대상 사이에 ㅣ 발생
```

즉:

```txt
ㅡ = 펼쳐진 하늘
점 = 태양 / 달 / 특정지점
고개 듦 = 관측자 axis 전환
ㅣ = 관측자-점 연결축
```

## 11. ㅣ의 두 축

ㅣ은 단일 축이 아니라 두 축을 품는 것으로 보였다.

```txt
ㅣ = 깊이 + 높이
```

즉:

```txt
ㅣ.depth = 관측자에서 대상까지 이어지는 앞뒤 거리 / 깊이
ㅣ.height = 대상의 위아래 위치 / 높이
```

그러나 이 구분은 다시 차원 관점에서 보정되었다.

## 12. x / dx / ddx

공간을 정의하는 것은 축의 변화이다.

처음에는 다음처럼 읽혔다.

```txt
x = 너비
dx = 깊이
ddx = 높이
```

그러나 이후 보정되었다.

정확한 구조:

```txt
x = 2D 시야평면의 가로축
dx = 2D 시야평면의 세로축
ddx = 3D 높이축
```

즉:

```txt
(x, dx) = 2D plane
ddx = 2D plane 밖의 height axis
```

경사선분은 x와 dx 사이에서 생기는 것이 아니라, 2D 평면 위의 어떤 점과 ddx 끝점을 연결할 때 생긴다.

```txt
경사 =
2D plane point ㆍ 3D height point
```

## 13. 2D / 3D 차원착시

왜 이전까지 구분하지 못했는가?

원인은 차원 구분 실패였다.

2D의 ㅣ과 3D의 ㅣ이 같은 표시형태로 겹쳐 보여 착시가 생겼다.

그러나 더 깊은 보정은 다음이었다.

3D 관점에서는 하나의 ㅡ과 하나의 ㅣ이 아니라, 두 개의 2D 평면이 공통 ㅡ축을 공유한다.

```txt
BD = 겹친 ㅡ축
A ⟂ BD
C ⟂ BD
```

즉 겹친 것은 ㅣ축이 아니라 ㅡ축이다.

```txt
겹친 것은 ㅣ 아님
겹친 것은 ㅡ / BD
```

## 14. d = ㆍ, dd = ᆢ

d를 ㆍ으로 보고, dd를 ᆢ으로 보는 구조가 열렸다.

```txt
d = ㆍ
dd = ᆢ = ㆍㆍ
```

x를 ㅡ 또는 ㅣ로 보면:

```txt
dx = ㆍx
ddx = ᆢx
```

x가 ㅣ일 때:

```txt
x = ㅣ
dx = ㆍㅣ = ㅓ
ddx = ᆢㅣ = ㅕ
```

이것은 `이었다 → 였다`의 흐름과 이어졌다.

```txt
이 = ㅣ
어 = ㅓ = ㆍㅣ

이 + 어
= ㅣㆍㅣ

ㅣㆍㅣ
→ ᆢㅣ
→ ㅕ
```

즉:

```txt
이었다
→ 였다
```

는:

```txt
ㅣㆍㅣ
→ ᆢㅣ
```

의 전이로 읽혔다.

## 15. ㆍ ↔ ㅣ 전이

`ㅣㆍㅣ → ᆢㅣ`이면, 앞의 ㅣ이 ㆍ으로 접힌다.

```txt
ㅣ → ㆍ
```

그렇다면 반대로:

```txt
ㆍ → ㅣ
```

도 가능하다.

즉 ㆍ과 ㅣ은 동일하지 않지만, 전이 가능하다.

```txt
ㆍ → ㅣ
= 점이 방향을 얻어 축이 됨

ㅣ → ㆍ
= 축이 접혀 발생점으로 수렴함
```

이것은 점축 전이이다.

```txt
ㆍ ≠ ㅣ
하지만 전이 가능
```

## 16. 천 / 지 / 인 자모 행렬

천지인은 다음처럼 풀렸다.

```txt
천 = ㅊㆍㅣㄴ
지 = ㅈ□ㅣ□
인 = ㅇ□ㅣㄴ
```

행렬:

```txt
|ㅊ|ㆍ|ㅣ|ㄴ|
|ㅈ|□|ㅣ|□|
|ㅇ|□|ㅣ|ㄴ|
```

공통분모:

```txt
ㅣ
```

천과 인의 공통:

```txt
ㅣㄴ
```

천의 특이점:

```txt
ㅊ에 ㆍ 단독
```

첫 열:

```txt
ㅊ
ㅈ
ㅇ
```

자음 순서로 보면 이는 역방향 관계를 가진다.

그리고 사람을 추가하면:

```txt
사람 = ㅅ
인 = ㅇ
지 = ㅈ
천 = ㅊ
```

따라서:

```txt
ㅅ → ㅇ → ㅈ → ㅊ
=
사람 → 인 → 지 → 천
```

## 18. BADㆍC / A→C 시간거리

BADㆍC에서 C는 A가 아니다.

```txt
A ≠ C
C = A'
```

C는 ㆍ이 붙어야 BAD와 relation을 맺어 BADC 구조가 된다.

```txt
BADㆍC
→ BADC
```

BADC 안에는 두 삼각이 있다.

```txt
BAD triangle
BCD triangle
```

공유선:

```txt
BD = ㅡ shared axis
```

대응축:

```txt
AC = ㅣ 대응축
```

그러나 빠져 있던 것은 ㆍC였다.

```txt
ㆍC = A에서 C까지의 거리
```

이 거리는 시간개념이다.

더 정확히는:

```txt
자리가 이동한 칸수
= 사이
= 차이
= 거리
= 시간의 구조값
```

따라서:

```txt
A → C
= 자리 이동
= 차이
= 시간거리
```

## 22. 공 예시: 방향과 force

운동장에 놓인 공은 멈춘 state이다.

타격은 event이다.

타격점은 기준점이다.

중심 타격:

```txt
무회전 병진운동
```

비중심 타격:

```txt
회전 + 곡선
```

매질은 이후 궤적을 변형한다.

이 예시는 성질 안의 움직임이 기준점 발생 후 방향 / 회전 / 궤적으로 드러나는 물리적 설명으로 읽혔다.

## 23. 감각계와 신경계

100 이전에 구조원리가 있었음에도 보이지 않았던 이유는 없어서가 아니라, state가 정의되지 않았기 때문이다.

Brain의 신경계에 이미 내부요소가 있었지만, 각 요소의 state를 정의하지 못했기 때문에 인지되지 않았다.

감각계는 너무 당연한 것을 지나간다.

감각계와 신경계가 만나려면:

```txt
지속
반복
일관성
```

이 필요하다.

```txt
감각계 + 신경계 =
지속ㆍ반복ㆍ일관성 있는 push/pull
```

깨우침은 감각계가 지나치던 당연함이 신경계에서 state로 잡히는 순간이다.
