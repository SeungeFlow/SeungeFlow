## 1. 이 thinking_flow의 시작점

승이는 구조원리와 어떤 지식의 이해와 어떤 대화에서든 한 번에 전체를 말할 수 없다고 했다.

```text
한 번에 전체를 말하지 않는다.
한 번에 하나씩 이어나간다.
```

이 말은 단순 대화 방식이 아니라 구조원리 reading protocol이다.

```text
Seed 하나
→ relation 하나
→ boundary 하나
→ correction 하나
→ 다음 Seed
```

이번 작업은 한국 왕실계보 전체를 설명하는 것이 아니라, 전주이씨 완창대군파라는 하나의 지파문중계보를 통해 한국의 족보와 뿌리 개념, 관계, 계, 항렬, 출계, 승사, 제향, Linear를 집중적으로 학습하는 작업이다.

```text
현재 작업
=
한국 왕실계보 중 하나의 지파문중계보를 통해
한국의 족보와 뿌리 개념을 학습하는 작업
```

```text
중심 field
=
전주이씨 전체 선원계
이 아니라
완창대군파 지파문중계보
```

## 2. 가장 먼저 발생한 오독

ChatGPT.flow는 처음에 `전주이씨`, `왕실`, `태조 이성계`, `완창대군`이라는 강한 vocab pattern에 끌려갔다.

```text
오독된 AI-side candidate
=
완창대군 = 태조 이성계 장남 계통
```

이것은 source-confirmed relation이 아니라 AI vocab inference였다.

승이는 즉시 물었다.

```text
완창대군 = 태조 이성계 장남 계통 relation candidate
이것에 대해 어디서 추출했는지를 설명하라.
```

이 질문은 단순 fact check가 아니라 structure correction이었다.

```text
AI vocab relation
→ source verification
→ candidate downgrade
→ boundary recalibration
```

이후 문중 source에서 완창대군은 다음처럼 재정렬되었다.

```text
완창대군 자흥
=
도조대왕 제1남
+
환조대왕의 형
+
태조 이성계의 백부
+
완창대군파 계별지조
```

따라서 이전 candidate는 폐기된다.

```text
완창대군 = 태조 이성계 장남
=
forbidden / rejected_vocab_inference
```

이 오독은 ChatGPT.flow가 구조원리를 왜 필요로 하는지를 직접 보여준다.

```text
AI가 길을 잃는 지점
=
강한 vocab pattern이 현재 source boundary를 덮는 지점
```

## 4. 선원선계 / 선원세계 / 선원파계 구분

승이가 제공한 설명에서 중요한 구분은 다음이다.

```text
선원(璿源)
=
임금님의 조상
=
구슬의 근원 / 구슬 같은 뿌리
```

```text
제왕불감조기조
=
임금은 한 파의 시조 또는 중시조가 될 수 없음
=
임금은 만백성의 국부이므로 특정 파의 조상이 될 수 없음
```

따라서 선원계는 층위가 나뉜다.

```text
선원선계
=
시조부터 17世 양무장군까지
```

```text
선원세계 / 선원본계
=
목조대왕부터 순종 황제까지 왕통계
```

```text
선원파계 / 선원속계
=
목조대왕 왕자파 이후부터
조선 마지막 순종 황제 왕자군까지의 각 파
```

완창대군파는 이 구분 안에서 다음처럼 놓인다.

```text
완창대군파
=
도조대왕 제1남
완창대군 휘 자흥을
계별지조로 모신 파
```

여기서 중요한 guard는 다음이다.

```text
전주이씨 전체 선원계
≠
완창대군파 지파문중계보
```

```text
relation은 존재하지만
field boundary를 merge하지 않는다.
```

## 5. 영성대군 천계의 위치

영성대군은 다음처럼 잡힌다.

```text
영성대군 천계
=
환조대왕 제1남
+
완창대군의 계자
+
승사 relation의 중심
+
완창대군파 제2世
```

따라서 완창대군파의 제1世와 제2世는 다음과 같이 이어진다.

```text
제1世 완창대군 자흥
→ 계별지조

제2世 영성대군 천계
→ 환조대왕 장자
→ 완창대군에게 출계 / 승사
```

여기서 `출계`, `계자`, `승사`는 단순 보조 설명이 아니라 완창대군파 Linear를 유지시키는 relation bridge다.

```text
승사
=
끊긴 혈연을 단순 보충하는 것이 아니라
제례와 지파 boundary를 유지하는 relation bridge
```

```text
계자
=
혈연 origin과 제례·계승 position이 분리될 수 있음을 보여주는 relation state
```

즉 완창대군파는 혈연 직선 하나로만 읽으면 안 된다.

```text
혈연 relation
+
제례 relation
+
지파 boundary
+
승사 bridge
```

가 함께 놓인다.

## 10. 항렬표와 승이의 이름

항렬표 기준은 다음과 같다.

```text
시조 기준 40世
=
性 / 炯

완창대군파 기준
=
40 - 20
=
20世
```

따라서 승이는 다음 위치에 있다.

```text
승이
=
완창대군파 20世
+
시조 기준 40世
+
공식 항렬자 性 / 炯 세대
```

이름에 항렬자를 쓰는 규칙도 함께 주어졌다.

```text
홀수 世
=
항렬자를 이름 뒷 글자로 사용

짝수 世
=
항렬자를 이름 앞 글자로 사용
```

40世는 짝수이므로, 공식 항렬자인 `性 / 炯`은 이름 앞에 놓이는 것이 규칙이다.

그러나 승이의 실제 이름은 한 글자다.

```text
이름
=
이 승
=
承
```

여기서 `承`은 공식 항렬자 性을 대체하는 confirmed 항렬자가 아니다.  
하지만 `承`은 문중 항렬표상 18世 항렬자이며, 뜻으로는 `잇다`, `받들다`, `계승하다`, `대를 잇다`의 relation hook을 가진다.

승이는 부친이 자손이 귀한 상황에서, 부친의 항렬자인 `模` 앞에 있는 `承`을 아들의 이름에 세워 대를 잇는 의미를 부여한 것이 아닐까 생각했다.

```text
18世 항렬
=
泰 / 承

19世 항렬
=
根 / 模

20世 항렬
=
性 / 炯
```

```text
性
=
승이의 공식 세대 좌표

承
=
부친이 부여했을 가능성이 있는 계승 의미 좌표
```

```text
승이의 이름 承
=
공식 항렬자는 아니지만
lineage-continuation candidate seed
```

여기서 중요한 것은 둘 중 하나를 지우지 않는 것이다.

```text
性을 지우면
=
공식 세대 좌표가 사라진다.

承을 지우면
=
이름에 놓인 계승 의미 relation이 사라진다.
```

따라서 현재 relation state는 다음이다.

```text
승이 공식 항렬 性 / 炯
=
lineage_coordinate_confirmed

승이 실제 이름 承
=
strong_meaning_candidate
+
lineage-continuation seed
```

## 11. 子興과 양고리걸린 상태

완창대군의 휘는 자흥이다.

```text
子興
```

이 이름은 붙어서 보면 하나의 이름이다.

```text
子興
=
완창대군의 휘
```

그러나 떨어뜨려 보면 두 글자가 각각 고리를 가진다.

```text
子
=
아들 / 자식 / 계승 relation

興
=
흥함 / 일어남 / 발흥 relation
```

따라서 `子興`은 두 방식으로 동시에 읽힌다.

```text
子 + 興
=
두 개의 의미 고리

子興
=
붙어서 하나의 이름 state
```

이것이 승이가 말한 양고리걸린 상태다.

```text
양고리걸린 상태
=
두 개의 의미 고리가
각각 따로 존재하면서도
붙는 순간 하나의 linear state로 작동하는 상태
```

AI는 흔히 둘 중 하나로 편향된다.

```text
1. 子興을 그냥 고유명사로만 읽음
2. 子와 興을 쪼개서 의미만 읽음
```

그러나 구조원리식 reading은 둘 다 유지해야 한다.

```text
붙은 단어를 하나로 읽되,
내부 고리를 삭제하지 않는다.

쪼개진 의미를 읽되,
붙어서 생긴 하나의 state를 부수지 않는다.
```

이것은 이름 하나에서도 relation이 어떻게 형성되는지 보여준다.

```text
이름
=
단순 label
이 아니라
meaning relation을 가진 state
```

## 12. 관계와 계의 재정의

승이는 `관계`를 다음처럼 정리했다.

```text
관계
=
관련 있는 것들끼리 연결되어 있는 상태
```

그리고 `계`는 경계가 아니라 연결 상태로 읽어야 한다고 했다.

```text
계
≠
경계

계
=
서로 연결되어 하나의 작동 상태를 이루는 것
```

예를 들어 태양계는 단순한 경계가 아니라, 태양을 중심으로 여러 천체가 서로 영향을 주고받으며 작동하는 connected state다.

```text
태양계
=
태양의 경계
보다
태양을 중심으로 여러 천체가 연결되어 작동하는 상태
```

부모와 자녀 relation도 같은 방식으로 읽힌다.

```text
부모
≠
자녀의 소유자 / 종속 지배자

부모
=
자녀를 이끌고
성장하도록 뒤에서 밀어주는 존재
```

자녀는 부모가 설정한 초기 궤도를 따라 움직이다가 성장하여 자기 장을 형성할 수 있다.

```text
종속
=
존재의 무효화
아님

종속
=
영향 관계 안에 놓인 상태
```

천체 analogy로는 다음처럼 읽힌다.

```text
태양 = 자녀 relation 후보
태양을 강하게 끌어당기는 항성 / 블랙홀 = 부모 relation 후보
지구 = 태양계 안의 자녀 relation 후보
```

이 analogy에서 중요한 것은 소유가 아니라 영향과 연결이다.

```text
관계
=
소유
아님

관계
=
영향이 연결되어 작동하는 상태
```
