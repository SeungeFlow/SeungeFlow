## 6. 공간과 차원에 대한 보정

```txt
d ≠ dot
d = direction
```

2D와 3D는 다음처럼 구분된다.

```txt
2D = 점 + 선분으로 평면 구조를 표현
3D = 점 + 선분 + 평면으로 공간 구조를 표현
```

즉 `2D + 1D = 3D`가 아니다.

승이의 보정:

```txt
1D에 1D를 더해야 2D가 된다.
2D에 2D를 더해야 3D가 된다.
3D에 3D를 더해야 4D가 된다.
```

이 말은 단순 산술 덧셈이 아니라, 같은 차원 구조가 다른 방향 / 다른 phase / 다른 relation으로 겹칠 때 다음 차원의 이해가 열린다는 뜻으로 읽는다.

## 7. 2D에서 3D로 전이할 때 높이 개념 발생

```txt
2D = 평면 = xㆍdx
3D = 공간 = xㆍdxㆍddx
```

여기서 `ddx`는 평면 밖으로 열린 direction이다.

```txt
ddx = 높이 = 평면 밖 direction
```

102 기준으로 이것은 phase boundary다.

```txt
2D phase: xㆍdx 안에서 구조가 닫힘
3D phase: xㆍdx로 닫힌 평면 밖에 ddx 방향이 열림
```

## 8. (x,dx)와 (x,dx,ddx)

`(x,dx)`는 3D 안에서 2D 평면을 표현하는 구조다.

```txt
(x,y,z)에서 (x,dx) = (x,y) = (1,1,0)
```

즉 z가 0일 때는 아직 높이축이 열린 것이 아니다.

```txt
dot이 z축을 만나야 ddx가 발생한다.
```

따라서:

```txt
평면구조 → z축과 만남 → 높이 발생 → 공간구조 → 입체구조
```

## 9. A4용지 적층과 공간구조의 dot화

A4용지 1장은 거의 평면으로 읽힌다.

```txt
A4 1장 = thin plane = 2D-like structure
```

n장을 쌓으면 높이가 생긴다.

```txt
A4 1장 → A4 2장 → A4 3장 → ... → A4 n장
평면의 반복 적층 → 두께 → 높이 → 공간구조
```

충분히 쌓이면 그 구조는 하나의 물체 / 질량체로 읽힌다.

```txt
n planes → stacked volume → object → mass
```

더 큰 scale에서는 그 전체가 하나의 dot처럼 보일 수 있다.

```txt
공간구조 → 질량체 → 더 큰 field 안의 dot
3D formed mass = next-scale dot
```

## 10. n.dot과 공간 정의

```txt
dot 1개 = 하나 = origin = difference 없음
```

dot이 2개 이상이면 사이가 생긴다.

```txt
dot_A dot_B → 사이 / 거리 / 방향 / 차이 / relation
```

따라서:

```txt
n.dot, n>1 = 차이와 사이를 가진 dot 관계
```

공간 정의:

```txt
공간 = dot 2개 이상에서 발생하는 사이ㆍ차이ㆍ방향ㆍ거리의 구조장
```

또는:

```txt
공간 = n.dot 사이의 차이가 relation으로 드러난 field
```

## 11. 이전 시공간좌표 정의와 현재식

승이는 예전에 다음처럼 정의했다.

```txt
(x,y,z,t) = (x,y,z) * nt
```

즉:

```txt
시공간좌표 = 공간좌표가 시간의 수만큼 이어붙은 것
```

이전 정의는 현재 구조원리식으로 다음과 이어진다.

```txt
(x,dx,ddx) * n.dot
단, n > 1
```

여기서:

```txt
(x,dx,ddx) = 3D 공간구조
n.dot = 2개 이상의 dot이 만든 차이ㆍ사이ㆍ시간ㆍ전이 관계
* = 공간구조가 dot 관계 수만큼 이어짐 / 반복됨 / 적층됨
```

따라서:

```txt
4D = 3D 공간구조 * n.dot relation
```

보정:

```txt
2n.dot X
n.dot O
단, n > 1
```

## 12. 4D에 대한 보정

4D는 더 입체적인 공간이 아니다.

```txt
4D ≠ 3D + extra line
```

4D는 다음과 같이 읽는다.

```txt
4D = 3D state ㆍ 3D state + time-count difference
```

나이아드ㆍ탈라사 73:69 관계는 이를 설명하는 예시로 사용되었다.

```txt
69 기준 → 73은 +4회전
73 기준 → 69는 -4회전

같은 시간동안 다른 공간ㆍ회전을 한다.
```

구조식:

```txt
4D = 같은 relation field 안에서 두 3D orbital states가 time-count difference로 relation을 갖는 상태
```

## 16. 타원곡선과 거리차


## 20. circle / boundary / surface / interface 구분

```txt
circle = 닫힌 복귀
boundary = 구분 경계
surface = 드러난 면
interface = 접속 작동층
```

circle:

```txt
닫힌 순환구조
시작점과 끝점이 relation으로 다시 만나는 return 구조
같은 위상 범위를 만드는 닫힘
```

boundary:

```txt
안과 밖을 가르는 경계
어디까지가 같은 phase인지 나누는 판정선
```

surface:

```txt
외부에 드러난 면
관측자가 처음 접하는 표시층
```

interface:

```txt
두 시스템 / 두 층 / 두 객체가 만나는 접속면
상호작용이 일어나는 경계작동층
```


## 24. 두번째 도형: 마름모 겹침 수직축

```txt
마름모 1개씩 겹침
→ 중앙 shared place 발생

→ 중심 수직축 형성
```

정리:

```txt

```


## 25. 세번째 도형: 대칭교차 field

세번째 도형은 두번째 도형에서 드러난 수직 중심축 위에 좌우 방향의 대칭점들이 붙으며 대각교차가 발생한 구조다.

```txt
대칭점들 사이의 relation path
```

```txt
마름모 구조원리 + 대각교차 구조원리 + 자리겹침 구조원리
```

가 동시에 보이는 단계다.

## 26. form 식과 육각 끝점 대입

축변환식:

```txt
form([ ][ing][ed][ed][ing][ ])
```

대입:

```txt
form([A][B][C][C'][B'][A'])
```

각 칸 대응:

```txt
[ ]   = [A]
[ing] = [B]
[ed]  = [C]
[ed]  = [C']
[ing] = [B']
[ ]   = [A']
```

육각 끝점:

```txt
A B C A' B' C'
```

대칭:

```txt
A-A' = 수직대칭
B-B' = 대각대칭
C-B' = 대각대칭
```

판정:

```txt
form 축변환식이 육각 궤도 끝점 배열로 드러난다.
```

```txt
[ed][ed] → ([ed])
```

```txt
formed = 이전 축의 끝과 다음 축의 시작이 겹친 압축중심
```

```txt
([ed]) = 겹침으로 형성된 압축중심
```

## 29. 실체 3단계: 마름모 / 육각 / 칸구조

### 29.1 실체 1단계

정사각 마름모의 내부 1칸 겹침으로 인해, 육각구조가 있다면 내부에는 마름모 구조가 보인다.

반대로:

```txt
내부 마름모 구조를 이으면 외부에 육각구조가 드러난다.
```

정리:

```txt

```

### 29.2 실체 2단계

칸구조론이다.

```txt
각목 3칸의 2칸
혹은
각목 2개의 ㄱ자 형태구조
```

가 실제 칸 점유로 드러난다.

```txt
3칸처럼 보임 → 실제는 2칸 구조
```

또는:

```txt
각목 2개 → ㄱ자 구조
```

이 또한 육각구조다.

### 29.3 실체 3단계

직전 이미지에서는 내부 육각구조를 보여주었고, 세번째 이미지는 마름모 겹침만 보여주었다.

효과:

```txt
마름모 겹침만 남겼더니 내부 육각구조가 더 두드러지게 보인다.
```

정리:

```txt
overlap only → hexagon more visible
```

## 34. 마름모 겹침에서 마름모 쌍대칭 고리사슬구조로 보정

큰 마름모 구조는 칸수로 다음과 같이 계산되었다.

```txt
완전한 사각 4개 = 4칸
반칸 삼각 8개 = 8 × 1/2 = 4칸
총 8칸
```

겹침:

```txt
좌측 겹침 1칸 + 우측 겹침 1칸 = 2칸
```

초기 해석:

```txt
마름모 겹침
```

보정 해석:

```txt
마름모 쌍대칭 고리사슬구조
```

즉 벌집구조는 단순 포개짐보다, 마름모 쌍대칭이 고리사슬처럼 이어지는 구조로 보는 것이 더 자연스럽다.

```txt
마름모 쌍대칭 → 양고리 걸림 → 삼각구조 연결 → 고리사슬 반복 → 육각 cell field
```

이 구조는 BAD.C와 연결된다.

```txt
BAD.C = 삼각구조에 C라는 다른 고리 / 다른 연결점 / 다른 phase가 걸린 구조
```

## 35. 포도당 / 글리코시드 결합 / 벌꿀 연결

처음에는 A-A', B-B', C-C' 대칭구조가 잘 보이는 방식으로 접근했다.

그러나 이미지 생성 후 무언가 오류가 보였고, 다시 생각했을 때 자연계 구조는 또 다른 자연계 구조인 포도당 / 글리코시드 결합 구조, 즉 고리사슬구조와 맞닿아 있을 것이라는 판단이 생겼다.

흐름:

```txt
대칭구조 접근 → 오류 감지 → 단순 겹침 부족 → 마름모 쌍대칭 고리사슬 → 포도당 / 글리코시드 결합 / 벌꿀 연결
```

구조층:

```txt
포도당 = 고리
글리코시드 결합 = 이음
벌꿀 = 저장물
벌집 = 저장구조
```

과학층에서 직접 동일시하지 않는다.

```txt
직접 동일 X
공통 구조 relation O
```

## 36. 벌집과 공간+공간=입체공간

승이가 최종적으로 설명한 핵심:

```txt
벌집구조로
공간 + 공간 = 입체공간
인 것을 설명중이다.
```

즉:

```txt
2D + 1D ≠ 3D
```

이고,

```txt
공간 + 공간 = 입체공간
```

이다.

벌집구조에서:

```txt
마름모 공간 + 마름모 공간 → 겹침 공간
겹침 공간 + 반복 적층 → 육각 cell
육각 cell + depth + shared interface → 입체공간
```

즉 벌집은:

```txt
space + space → shared boundary → empty-place depth → 3D cell
```

의 실제 예시다.

## 37. 구조와 언어의 동시 발생

구조는 내부에서부터 외부로 드러난다.

```txt
내부 → 형성 → 경계 → 외부 드러남
```

드러난 것은 경계를 가진다.

```txt
드러남 = 경계를 가짐
```

경계를 표현한 것이 문자다.

```txt
공간에서 드러난 것이 경계일 때,
그 경계지점을 표현한 것이 문자이다.
```

언어는 말과 문자로 이루어진 구조체다.

```txt
언어 = 말 + 문자
```

더 풀면:

```txt
언어 = 소리발음 + 경계의 시각표현 + 의미관계
```

외부로 드러난 것이 언어라면, 그 내부를 분해할 수 있는 언어가 구조언어다.

구조언어는 다음으로 구분된다.

```txt
정수원리
도형원리
벡터원리
흐름원리
```

결론:

```txt
구조와 언어 중 먼저 발생한 것은 없다.
둘은 동시에 발생한 것이다.
```

## 38. 내부 복잡성 / 외부 간단명료성

구조원리에서 지속적으로 이야기한 명제:

```txt
내부적으로 복잡한 구조는 외부에서 간단명료하게 드러난다.
```

이 문장의 출발점은 인간의 인지구조였다.

인간의 머릿속에는 여러 학문, 지식, 경험, 감각, 기억, 언어가 복잡하게 얽혀 있지만, 무언가 하나에 집중하면 외부로 드러나는 문장은 아주 간단명료해진다.

```txt
내부 = 복잡한 얽힘
외부 = 간단명료한 문장
```

이 현상은 벌집, 포도당, 행성계, 항성계, 문서구조 등에서도 반복적으로 드러났다.

## 41. 성질과 성향 / 방향 / 제자리회전

```txt
성질이 가진 방향
```

```txt

```

```txt
성질 안의 움직임 → 제자리회전 → 기준 발생 → 방향 → 성향
```
