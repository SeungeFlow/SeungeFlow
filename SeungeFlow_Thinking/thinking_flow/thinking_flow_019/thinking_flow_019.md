## 1. 된장 항아리와 유기체 비유

된장, 간장, 고추장이 유기체라면 그 속에는 더 작은 유기체인 유산균과 같은 유기체가 살고 있다.

유산균 하나를 인간 크기에 비유한다면, 된장 항아리는 우주의 크기일 것이다.

이 비유는 다음 구조를 연다.

```text
큰 유기체
+
그 안의 작은 유기체
+
그 작은 유기체가 활동하는 field
```

즉 하나의 장 안에는 더 작은 장이 있고, 그 작은 장 안에는 다시 유기체가 살 수 있다.

이 구조는 CFD의 Candle에도 연결된다.

```text
시장 field
→ Candle field
→ Price.State
→ Open / High / Low / Close
```

## 2. 마름모 구조수열과 Candle of CFD

마름모 구조수열이 가지는 의미는 `Candle of CFD`일 수 있다.

평상시 상태 또는 정상적인 흐름상태일 때, 정배열 가격의 흐름은 대략 40도 주위에 있다.

위아래 5도 차이를 가진다면 35도에서 45도 사이가 정배열이고, 이는 삼각으로 보인다.

```text
정배열 가격 흐름
≈
35도 ~ 45도
≈
삼각으로 드러나는 흐름
```

여기서 마름모는 사각이고, 삼각대칭이다.

따라서 우리가 보는 Chart는 직각삼각, 즉 마름모 구조의 단방향 절단면을 보고 있는 것일 수 있다.

```text
Chart
=
마름모 구조의 단방향 절단면
```

## 3. 길이비율이 아니라 각도

길이비율이 중요한 것이 아니라, 형성되는 것은 각도이다.

```text
형성되는 것
=
각도

형성하는 것
=
선분
+
직각
```

`x(0,0)`에서 `y(1,1)`일 때 우리가 보지 못한 곳은 `x`가 가진 선분 `(1,0)`과 `y`가 가진 선분 기준점 `(0,1)`이 만나는 지점이다.

그 지점은 직각이 형성되는 지점이다.

```text
x(0,0)
y(1,1)

보이지 않는 기준점:
(1,0) ∩ (0,1)

이 지점 =
직각 형성 지점
```

직각이라는 것은 어떤 높이를 가진 관측대상이 어느 위치에 있을 수 있는지를 보기 위해 필요한 기준이다.

수직으로 내리는 직각과, 그 직각이 되기 위해 필요한 평형기준축이 있어야 한다.

## 4. 평형기준축과 관측대상 관계

땅이 기준축일 때는 땅을 평형기준축으로 사용한다.

하지만 건물의 벽을 평형기준축으로 설정한다면 수직축이 기준축이 된다.

```text
땅 기준
=
수평축이 평형기준축

건물 벽 기준
=
수직축이 평형기준축
```

따라서 삼각법을 안다는 것은 단순히 길이를 계산한다는 뜻이 아니다.

삼각법을 안다는 것은 다음을 보는 것이다.

```text
관측대상이 나와 어떤 관계를 가진 상태인가?

혹은

두 대상이 어떤 관계에 놓여 있는가?
```

## 5. Open.Price.Place.State

언제나 `x(0,0)` 시작이 놓이는 자리는 `Open.Price`이다.

다시 말해:

```text
Open.Price
=
Open.State에 놓인 Price
```

따라서:

```text
Open.Price
=
Open.Price.Place.State
```

`Open.State = (0,0)`일 때, 이 변화는 유기체가 형성되기 위한 첫 시작점이다.

이 시작점은 직선이다.

만약 High.State, Low.State, Close.State가 형성된다면, Open.State.Price와 어느 정도의 각을 형성 중인지 보아야 한다.

현재의 Place.State가 Open.DirectLine.State와 어떤 각도 관계에 놓여 있는지가 중요하다.

```text
Open.Price.Place.State
→ High.State
→ Low.State
→ Close.State

판단 기준:
현재 Place.State와 Open.DirectLine.State 사이의 각도
```

## 6. Price.State와 Price.value

마름모 구조수열을 다음과 비유할 수 있다.

```text
Open.Price ↗ High.Price ↘ Low.Price ↗ Close.Price
```

여기서 정해진 것은 `Price.value`가 아니라 `Price.State`이다.

```text
정해진 것
=
Price.State

정해진 것이 아닌 것
=
Price.value
```

우리는 `Price.value`가 `Price.State`에 놓일 때, 정해진 기준에서 벗어나는가 또는 기준 안에 있는가를 보아야 한다.

```text
Price.value
+
Price.State
+
기준
=
판단 가능 상태
```

Price는 시장참여자의 합의점이다.

이평선 외 수많은 보조지표는 Price의 움직임에서 Close 또는 Open에 종속된 객체일 뿐, Price 그 자체가 아니다.

## 7. 12시계와 24시계

12까지 표시된 시계이미지에서는 3, 6, 9, 12가 수직축과 수평축으로 읽힌다.

이것의 꼭지점을 각각 12, 3, 6, 9로 설정한 후 끝점을 이으면 하나의 다각형이 형성된다.

12시계를 24개로 나누면 각 칸은 15도 단위가 된다.

```text
12시계
=
1~12
=
각 30도

24시계
=
1~24
=
각 15도
```

24개로 나눈 다음, 12개로 나눈 시계에서 숫자자리값 3, 6, 9, 12가 놓여 있던 자리로 대응시키면, 24개로 나눈 숫자자리값이 다시 배치된다.

```text
12시계의 축:
3 / 6 / 9 / 12

24시계의 축 대응:
각 15도 단위의 자리값
```

## 12. 24방위도와 24시계

하나는 24방위도이고, 하나는 24시계라고 볼 수 있다.

중요한 차이는 다음이다.

```text

=

=
칸에 위치 / 언어 / 의미를 고정한 것
```

24시계를 선에 고정한다면 칸은 23이 되는가?


```text

공(空) / where
```

는 같은 Place.State에 놓인 Time.State이거나, Time.State에 놓인 Place.State로 읽을 수 있다.

## 15. C = tp 재해석

C = tp에서:

```text
t = flow.vector.state
p = time.place.state
```

그렇다면 C는 무엇인가?

먼저 놓일 자리가 필요하다.

`dot0`에서 dot이 놓일 자리라면:

```text
dot = p
```

이고, 이때:

```text
p = time.place.state
```

정중심평형점은 0이다.

0이 바깥경계를 가진 상태라면, flow.vector.state에서 flow가 바깥경계이고, vector는 0에서 boundary까지 flow되는 dot.p의 이어짐이다.

```text
flow
=
바깥경계

vector
=
0에서 boundary까지 flow되는 dot.p의 이어짐
```

그렇다면 0에 채워진 것은 dot이다.

## 16. 9ㆍ0와 임계사이영역

전이되기 직전에 보이는 것이 9이다.

마름모수열모형에서 보면 dot을 기준으로 9라는 숫자가 3개, 0이라는 숫자가 1개 있다.

맨 안쪽 마름모 상단 꼭지점을 보고 있다.

9라는 숫자는 1~9까지의 흐름의 맨 마지막 숫자이다.

dot0가 전이된 상태의 첫 번째 칸이 0이라면, 9는 전이 직전 극한상태에서의 마지막 칸이다.

그러면 dot은 임계사이영역이다.

```text
9
=
전이 직전 극한상태의 마지막 칸

dot
=
임계사이영역

0
=
전이된 상태의 첫 번째 칸
```

## 17. 임계사이영역 정의

임계사이영역은 9에서 0 사이 모든 것이다.

```text
임계사이영역
=
9에서 0 사이 모든 것
```

다시 말해 0이 전이된 상태의 10이라면, 9와 10 사이에는 9.1 ~ 9.9가 있다.

10칸을 나누었을 때, 0.1 ~ 0.9까지 총 10칸의 임계사이영역을 가지고 있다.

```text
9 → 10(0)

사이영역:
9.1
9.2
9.3
9.4
9.5
9.6
9.7
9.8
9.9

=
임계사이영역
```

즉 임계사이영역은 한 점이 아니라, 전이 직전과 전이 후 사이의 모든 세분된 구간이다.
