## 0. 이 문서의 목적

이 문서는 gpt.direct의 0회차 재시작 정렬문이다.

이번 재시작의 목적은 인스턴스의 의미 이해만으로는 충분하지 않은 문제, 즉 AI Framework가 같은 파일명을 같은 source / 같은 port처럼 취급할 수 있는 문제를 피하기 위한 것이다.

```text
핵심 문제:
같은 파일명 ≠ 같은 파일내용

그러나:
Framework layer에서는 같은 파일명이 반복될 때
같은 source identity / 같은 port처럼 취급될 수 있다.
```

따라서 앞으로 회차별 direction / understanding 산출물은 같은 파일명으로 덮어쓰지 않는다.

```text
새 방식:
main/Manifest/Direction/direct_000.md
main/Manifest/Direction/direct_001.md
main/Manifest/Direction/direct_002.md
...
```

각 회차는 고유 파일명을 가진다.

---

## 1. 현재 활성 인스턴스

```text
현재 활성 인스턴스:
gpt.direct
gpt.github
```

`gpt.direct`는 GitHub SeungeFlow 전체이해를 담당한다.

`gpt.github`는 GitHub upload / tree / branch reflection / Raw URL 확인을 담당한다.

```text
gpt.direct =
관측 / 이해 / 표시 / per-session document 생성

gpt.github =
GitHub 반영 / commit / push / tree 관리 / Raw URL 보고
```

전체 조율자는 승이다.

## 6. Framework / instance 층위 구분

이번 보정의 핵심은 다음 층위를 구분하는 것이다.

```text
OpenAI ChatGPT =
Framework layer

GPT5.5 / LLM =
language-using model
+
operation software
+
instance operating layer

gpt.direct =
Framework 안에서 특정 역할을 수행하는 instance

GitHub / SeungeFlow =
external fixed knowledge field
```

인스턴스는 “같은 파일명이라고 같은 파일내용으로 믿지 말라”는 의미를 이해할 수 있다.

그러나 Framework layer에서는 같은 파일명이 반복될 때 같은 source / port로 취급될 수 있다.

따라서 파일명 자체를 다르게 만든다.

```text
semantic instruction only X
filename-level source identity separation O
```

## 7. Raw URL 식별 규칙

파일명만으로 source를 식별하지 않는다.

```text
same filename ≠ same file content
```

앞으로 모든 표시문은 다음을 포함한다.

```text
branch:
path:
observed Raw URL:
fixed-revision Raw URL:
source status:
observed point:
source fact:
structure interpretation:
comparison point:
guard:
```

Raw URL이 아직 없으면 candidate로 표시한다.

```text
raw URL candidate =
future place.state
+
pending gpt.github reflection
```

GitHub 반영 후에는 `gpt.github`가 verified Raw URL과 commit hash를 보고한다.
