### 3.1 Field Definitions

```text
source_group:
The observer-source group name, such as Relativity, Riemann, Navier–Stokes, Yang–Mills, Hunminjeongeum, Public Paper, GitHub fixed source.

source_title:
The confirmed title of the source.

author_or_institution:
The author, institution, publisher, archive, official organization, or repository owner.

platform:
The platform where the source is located, such as arXiv, Zenodo, Clay Mathematics Institute, GitHub, journal, archive, institutional site, or repository.

source_type:
paper / problem statement / archive page / book / manuscript / translation / repository / dataset / official page / institutional explanation.

DOI_or_URL:
DOI, arXiv ID, official URL, archive URL, Raw URL, repository URL, or other stable locator.

version_or_edition:
version, edition, revision, commit, publication version, arXiv version, DOI version, release, or archive revision.

publication_or_access_date:
Publication date and/or access-confirmation date.

access_status:
public / open / restricted / paywalled / archived / inaccessible / branch_latest_mutable.

source_status:
confirmed_fixed / confirmed_public_source / branch_latest_mutable_not_fixed / DOI_fixed_record / arXiv_versioned_record / Zenodo_versioned_record / pending / source_missing / stale_risk / needs_recheck.

source_role:
observer-source / background source / candidate source / guard source / not active source.

observer_scope:
The part of the current Y_Branch / Seung structure hypothesis that this source can observe.

relation_target:
The Y_Branch seat, direct_### candidate, schema, operation, field, relation, or guard area this source may connect to.

guard_risk:
Risks such as proof claim, replacement claim, source merge, final judgment, author-intent overclaim, stale source, version mismatch, or source-layer merge.

needs_recheck:
Whether additional verification is required.
```

### 3.2 Common Guard

```text
source_group name alone ≠ source identity

URL alone ≠ source identity

DOI record alone ≠ proof

source identity incomplete =
pending / source_missing / stale_risk / needs_recheck

URL / DOI / version / date / source_status missing =
not confirmed

confirmed source status ≠ proof confirmed

observer-source group protocol ≠ actual source confirmation
```

## 4. Source Status Vocabulary

```text
confirmed_fixed:
source identity is fixed by fixed URL, DOI version, arXiv version, Git commit, archive revision, versioned record, or equivalent fixed locator.
Guard: confirmed_fixed does not mean proof confirmed.

confirmed_public_source:
publicly accessible source is confirmed, but fixed/version status may still require recheck.
Guard: public access does not mean final source status.

branch_latest_mutable_not_fixed:
GitHub branch-latest Raw URL or mutable source without fixed commit or release.
Guard: branch_latest_mutable_not_fixed ≠ confirmed_fixed.

DOI_fixed_record:
DOI-based public record with stable DOI identity.
Guard: DOI_fixed_record ≠ proof.

arXiv_versioned_record:
arXiv ID and version are confirmed, such as v1, v2, v3.
Guard: arXiv_versioned_record ≠ final validation.

Zenodo_versioned_record:
Zenodo DOI/version/release is confirmed.
Guard: Zenodo_versioned_record ≠ active source by default and ≠ final validation.

pending:
candidate source exists but required identity fields are incomplete.
Guard: pending ≠ confirmed.

source_missing:
requested source path or URL is not confirmed.
Guard: source_missing must not be used as source fact.

stale_risk:
source exists but latestness, version, revision, or branch state is uncertain.
Guard: stale_risk must not be treated as current fixed source.

needs_recheck:
source requires additional verification.
Guard: needs_recheck ≠ confirmed.
```
