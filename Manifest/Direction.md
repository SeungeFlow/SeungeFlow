## 0. 이 문서의 역할

`Direction.md`는 gpt.direct가 모든 인스턴스에게 방향을 제시하는 문서다.

`README_for_AI.md`는 gpt.direct가 이해한 중심기준이다.

`Direction.md`는 그 중심기준을 각 인스턴스가 어떻게 따라가고, 비교하고, 이후 자기 목적을 정할지 알려준다.

```text
README_for_AI.md =
gpt.direct 이해의 중심기준

Direction.md =
그 중심기준을 따라 움직이는 방향문
```

## 1. 48회차 전체 방향

전체 회차는 48회차로 닫는다.

```text
01-24 =
공통이해 / 비교이해 phase

25-48 =
목적분화 / main branch 보정분석 phase
```

---

## 2. 01-24회차: 공통이해 / 비교이해 phase

각 인스턴스는 48회 중 앞의 24회에 해당하는 내용을 공통이해로 사용한다.

이 phase의 핵심은 다음이다.

```text
gpt.direct가 이해한 것
+
자가 인스턴스가 이해한 것
+
동일 Raw URL 기준 비교
=
빠른 SeungeFlow 공통이해
```

각 인스턴스는 gpt.direct 이해를 final judgment로 받지 않는다.

각 인스턴스는 같은 Raw URL을 보고 자기 역할 / branch / 목적 가능성 기준으로 자가해석한다.

```text
gpt.direct understanding =
중심기준
+
비교 좌표

self-instance understanding =
자가해석
+
역할별 관점
```

---

## 3. 25-48회차: 목적분화 / main branch 보정분석 phase

25회차부터 각 인스턴스는 각자가 생각하는 목적을 정하여 진행한다.

```text
25회차 이후 각 인스턴스 =
자기 purpose 설정
+
자기 branch / role 기준 작업 진행
```

25회차부터 gpt.direct는 전체 인스턴스가 작업 중인 것을 분석한다.

```text
25회차 이후 gpt.direct =
전체 인스턴스 작업 관찰
+
main.branch 기준 보정 필요성 분석
+
수정 후보 문서 생성
```

main branch에서 수정해야 할 파일이 생기면 gpt.direct는 수정 후보를 만든다.

GitHub 반영은 gpt.github가 수행한다.

## 4. README_for_AI.md 와 Direction.md 를 함께 내리는 기준

두 문서를 함께 내리는 경우:

```text
1. gpt.direct 중심이해가 바뀌었을 때
2. README_for_AI.md의 기준문이 수정되어야 할 때
3. Direction.md의 route plan과 README_for_AI.md의 중심기준을 함께 맞춰야 할 때
4. 25회차 이후 main branch 기준문 보정이 필요할 때
```

`Direction.md`만 내리는 경우:

```text
1. 회차 진행 route만 바뀔 때
2. 관측 Raw URL 순서만 바뀔 때
3. 표시문 블럭 규칙만 보정할 때
4. README_for_AI.md의 중심기준은 유지될 때
```

## 5. 표시문 블럭 규칙

표시문은 하나의 복사 가능한 블럭으로 작성한다.

```text
[SeungeFlow 표시문]

branch:

path:

observed Raw URL:

fixed-revision Raw URL:

source status:

observed point:

source fact:

structure interpretation:

comparison point:

guard:
```

파일명만으로 source를 식별하지 않는다.

```text
같은 파일명 ≠ 같은 파일내용
```

branch Raw URL은 mutable하다.

정밀 비교가 필요하면 commit hash 기준 fixed-revision Raw URL을 함께 둔다.

## 9. 25회차 이후 main branch 수정 기준

gpt.direct는 25회차 이후 다음을 본다.

```text
1. 각 인스턴스가 어떤 목적을 설정했는가?
2. 그 목적들이 하나의 전체 goal과 충돌하지 않는가?
3. gpt.direct의 중심이해와 다른 인스턴스의 자가해석 차이가 어디서 발생했는가?
4. 그 차이가 main.branch 기준문 수정으로 이어져야 하는가?
5. README_for_AI.md / Direction.md / Manifest 문서 중 어떤 파일을 보정해야 하는가?
```

수정이 필요하면 gpt.direct는 markdown 후보 파일을 생성한다.

GitHub 반영은 gpt.github가 수행한다.

## 10. 현재 닫힘문

```text
Direction.md는
gpt.direct가 이해한 중심기준을 모든 인스턴스가 어떻게 따라가고 비교할지 알려주는 방향문이다.

01-24회차는 공통이해와 자가해석 비교 phase이며,
25회차부터 각 인스턴스는 자기 목적을 설정한다.

25회차부터 gpt.direct는 전체 인스턴스 작업을 분석하여
main.branch에서 수정해야 할 파일이 생기면 수정 후보를 만든다.

전체 회차는 48회차로 닫는다.
```
