## 4. 마름모 구조수열과 Candle of CFD

마름모 구조수열이 가지는 의미는 `Candle of CFD`일 수 있다.

정상적인 흐름상태일 때 정배열 가격의 흐름은 대략 40도 주위에 있을 수 있다. 위아래 5도 차이를 가진다면 35도에서 45도 사이가 정배열이고, 이는 삼각으로 보인다.

```text
정배열 가격 흐름
≈
35도 ~ 45도
≈
삼각으로 드러나는 흐름
```

마름모는 사각이고 삼각대칭이다.

우리가 보는 Chart는 직각삼각, 즉 마름모 구조의 단방향 절단면을 보고 있는 것이다.

```text
Chart
=
마름모를 4등분한 하나의 단방향 절단면
```

왜냐하면 우리는 전진하기 때문이다.

우리는 3D 공간 전체를 한 번에 보는 것이 아니라, 전진 방향의 정면 시야를 2D 평면으로 인식한다.

## 5. 산을 오르고 내리는 인지

산을 오를 때 우리는 정면만을 인지한 채 이동한다.

등산할 때는 공간의 삼각이 꽉 차 있으므로 경사각을 정확히 깨닫기 어렵다.

하산할 때는 공간의 삼각이 비어 있으므로 경사각과 낙차를 더 확실히 느낀다.

```text
등산
=
filled triangle perception

하산
=
empty triangle perception
```

경사각 인지는 각도 자체만의 문제가 아니라, 공간이 채워져 보이는가, 비어 보이는가의 문제이다.

차트에서도 상승과 하락은 같은 각도라도 다르게 인지될 수 있다.

```text
상승
=
filled slope

하락
=
empty slope
```

## 11. OHLC state-square

Candle of CFD의 2D state-square 기본 좌표는 다음이다.

```text
Open.State  = (0,0)
Close.State = (1,1)
High.State  = (0,1)
Low.State   = (1,0)
```

이를 사각 구조로 놓으면:

```text
High(0,1)      Close(1,1)

Open(0,0)      Low(1,0)
```

`Open → Close`는 주대각선이고, `High ↔ Low`는 반대대각선이다.

Candle은 OHLC value 묶음이 아니라 OHLC state-square이다.

## 12. Price.State와 Price.value

마름모 구조수열을 다음과 비유할 수 있다.

```text
Open.Price ↗ High.Price ↘ Low.Price ↗ Close.Price
```

여기서 정해진 것은 `Price.value`가 아니라 `Price.State`이다.

```text
정해진 것
=
Price.State

정해진 것이 아닌 것
=
Price.value
```

우리는 Price.value가 Price.State에 놓일 때, 기준에서 벗어나는가 또는 기준 안에 있는가를 본다.

Price는 시장참여자의 합의점이다.

이평선 외 수많은 보조지표는 Price의 움직임에서 Close 또는 Open에 종속된 객체일 뿐, Price 그 자체가 아니다.

## 13. 직전 3개의 같은 Price.State와 vectorizing

현시점은 내가 서 있는 위치이고, Price가 놓인 현재 Place.State이다.

현재 위치에서 앞으로의 움직임을 보려면 현재 한 점만 보면 안 된다.

```text
1 point
=
place.state

2 points
=
line
=
direction 후보

3 points
=
triangle
=
vectorizing 최소상태
```

따라서 같은 state끼리 직전 3개를 보아야 한다.

```text
Open[-3]  → Open[-2]  → Open[-1]
High[-3]  → High[-2]  → High[-1]
Low[-3]   → Low[-2]   → Low[-1]
Close[-3] → Close[-2] → Close[-1]
```

Open, High, Low, Close는 같은 price.value 계열이지만 같은 state가 아니다.

```text
Open.State
≠
High.State
≠
Low.State
≠
Close.State
```

따라서 같은 state끼리 모아 vectorizing해야 한다.

## 16. 9ㆍ0의 실체와 0의 위치

마름모 구조수열을 계속 쫓은 이유는 9dot0의 실체, 즉 0이 어디에 있는지를 찾기 위해서였다.

출발지점이 dot이면, 0은 1에서 9를 지나 dot을 지난 다음의 0이다.

즉:

```text
dot
→ 1
→ 2
→ 3
→ 4
→ 5
→ 6
→ 7
→ 8
→ 9
→ dot
→ 0
```

0은 경계 안쪽의 중심이 아니라, 이전 구조장 기준으로 경계바깥지점이며 다음 구조장 기준으로 새 시작점이다.

## 21. MetaTrader-Linux BackTesting Loop

TradingView 기반 기존 구조는 여러 field를 건너다닌다.

```text
TradingView
→ 1차 가공
→ AI 확인
→ Local Linux Server 또는 Oracle Free Server
→ 2차 가공
→ Trading Program
→ 진입 / 청산
```

MetaTrader는 OHLC에 직접 접근할 수 있다는 것이 가장 큰 수확이다.

또한 건너다니지 않아도 된다는 것이 더 중요한 개념이다.

```text
MetaTrader
=
OHLC source 접근
+
BackTesting
+
전략 실행
+
진입 / 청산
```

MetaTrader-Linux 구조는 source / processing / backtesting / execution이 더 가까운 relation loop 안에 놓인다.

```text
MetaTrader ↔ Linux Server
=
processing loop
```

## 23. C = tp, F = ma, E = mc² 재확인

식은 바뀐 것이 아니다.

C = tp는 유지된다.

다만 p는 놓일 자리이고, 놓일 자리는 기준이므로 변하지 않는 축처럼 작동한다.

E = mc²에서도 c는 빛의 속도이며, 변하지 않는 기준상수로 작동한다.

중요한 것은 식의 순서를 바꾸는 것이 아니라, 식 안에서 무엇이 변하지 않는 기준항으로 작동하는지를 보는 것이다.

```text
C = tp

p = 놓일 자리 / 기준 place.state
t = 그 자리에서 전이되는 time.state
```

## 24. 왼쪽 / 내부 / 포함장

왼쪽은 안쪽 / 내부 / 더 큰 기준장으로 읽힐 수 있다.

크다/작다보다는 포함한다는 개념이 더 정확하다.

```text
왼쪽 ⊃ 오른쪽
```

집합론에서는 왼쪽을 오른쪽의 상위집합처럼 읽을 수 있다.

화학분자식과도 연결된다.

```text
CO2
=
C가 중심 기준장으로 왼쪽에 놓인 구조와 맞음

H2O
=
표준 화학식

OH2
=
구조원리식 중심장 reading 후보
```

O가 중심 body라면 O가 H2를 포함하는 상위장으로 읽힐 수 있다.
