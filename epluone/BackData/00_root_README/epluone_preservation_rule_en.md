# epluone preservation rule

```text
phase = 3/4
document = epluone_preservation_rule_en.md
role = preservation rules
```

## 0. Highest Principle

```text
no delete
only add
```

This rule applies to file preservation, document interpretation, and directory operation.

---

## 1. Forbidden

```text
Do not delete original sources.
Do not create one integrated summary.
Do not merge all material into one document.
Do not preserve only AI output while deleting Seung input.
Do not delete Stop / Next / Flow.
Do not delete failure records.
Do not delete Break Test collapse records.
Do not promote CFD to core theory.
Do not reduce sohosa_hyangsa to a simple application.
```

---

## 2. Allowed

```text
Indexing
Directory placement
Source maps
Relation maps
README files
Separation of source and interpretation
Candidate archives
```

---

## 3. Document Handling Principles

```text
Original source
=
source in itself

Interpretation
=
separate interpretation file

Summary
=
allowed only as support and cannot replace source

README
=
entrance document

relation map
=
map

manifest
=
placement table
```

---

## 4. AI Output Handling Principles

```text
AI output may be a source.
However, it must be separated from Seung input.

AI output is not final truth.
It is interpretation trace, response layer, or verification candidate.
```

---

## 5. Failure Record Handling

```text
Failure
=
not deletion target

Collapse
=
material for the next structure

Stop
=
critical point just before coordinate collision

Break Test
=
weak-link discovery device
```

---

## 6. Anti-Compression Rule

```text
Too much compression erases meaning.
Too much summarization erases structure.
Too much arrangement can kill flow.
```

Compression is allowed only for:

```text
search index
manifest for placement
guide for reading order
```

---

## 7. Final Compression

```text
In epluone,
original source comes first,
interpretation comes next,
summary is only support,
and deletion is not used.
```
