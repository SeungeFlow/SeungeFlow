# META: 000_dot

## 0. document character

`000_dot.meta.md` is a shared structure file.

This document defines `dot` not as a visual point, a mathematical value, or a fixed symbol, but as the first minimal place-state in which existence can be placed as an observable target inside an Active.Schema field.

This document does not replace all later dot-like functions.

This document is an origin pointer schema: it preserves the boundary of `dot` while pointing to dot-like functions that appear in later schema documents.

---

```text
dot

the first minimal place-state
secured inside a field

```

## 2. dot and existence

Existence is not created by dot.

Rather:

```text
dot
=
the place condition
that allows existence to be observed as placed
```

Thus:

```text
existence can be seated
→
field becomes readable
→
relation can begin
```

A dot is the beginning of readable placement, not the whole object.

## 3. dot and relation

One dot alone is not relation.

```text

→
```

```text

```
