P_place =

6. 약한 고리 6 — 수학 해석과 수학 증명 혼동

2단계 전체에서 계속 통과 판정을 내렸지만, 그 통과는 Ctp 구조 대응 통과다.

위험 표현

따라서 Collatz를 증명했다.

Kakeya를 Ctp로 해석했다.
따라서 Kakeya conjecture를 증명했다.

Hadamard를 Ctp로 정리했다.
따라서 Hadamard conjecture를 증명했다.

전부 실패다.

정식 구분
Ctp 구조 대응:
문제의 단어·수학식·코드형 표현이
구조원리와 같은 구조를 가리킨다.

수학 증명:
해당 수학 명제가 기존 수학의 엄밀한 기준으로 참임을 보인다.
판정
현재까지의 1~2단계 결과:
Ctp 구조 대응 통과

아직 아닌 것:
각 문제의 일반 수학 증명 완료

8. 추가 약한 고리 — ?를 뒤에 두는 습관

3단계 2회차에서 정리한 역검산은 중요하다.

정방향:
C = Ctp(t, P_place, m, ?)

역검산:
? → m → P_place → t → C
위험 표현
먼저 t를 정한다.
그다음 m을 끼워 넣는다.
마지막에 ?를 붙인다.

이러면 해석이 흔들린다.

정식 절차
1. ? 전제조건과 사전정의를 세운다.
2. m 관측대상을 정한다.
3. P_place 놓일 자리장을 확인한다.
4. t 관계차/전이차를 판독한다.
5. C가 field로 닫히는지 본다.
약한 고리

?를 마지막에 두면, 다음 문제가 생긴다.

확률 p와 P_place 충돌
m의 층위 혼동
시점/지점 혼동
관측기준 변화 누락
기준축 변화 누락

그래서 3단계 이후는 반드시 역검산으로 간다.

5회차 판정 근거자료
1. 사이와 차이의 층위 혼동

근거는 1단계 결과문에 직접 있다.

문서는 다음처럼 구분한다.

사이 = 객체 간 관계가 형성되는 조건
차이 = 관계와 관계 사이의 거리차 / 스케일차 / 상태차
임계사이영역 = 차이가 구조출현으로 넘어가는 전이장

그리고 더 명확히:

임계점 = 대표 좌표
임계사이영역 = 실제 작동장
차이는 판독값이고, 사이는 전이가 일어나는 장이다.

따라서 사이를 곧바로 차이로 읽거나, 임계사이영역을 임계차이영역으로 바꾸면 층위가 무너진다. 이게 첫 번째 위험지점이다.

2. dot / ㆍ의 과잉 동일시

논문은 000_dot을 매우 조심스럽게 정의한다.

dot은 값도 아니고, 0도 아니고, ㆍ도 아니고, 원도 아니며, CoreDot도 아니다.
dot은 존재가 존재할 수 있도록 field 안에 처음 확보되는 최소 place-state이다.

또 dot은 존재 자체가 아니라, 관측대상이 field 안에 놓일 수 있는 최초 자리라고 말한다. 하나의 dot만으로 relation이 발생하지 않고, relation은 최소 두 자리 사이의 차이에서 시작된다고 되어 있다.

즉 dot = ㆍ = 존재 = 벡터씨앗 = 전자 = 원 = 0처럼 한꺼번에 동일시하면 무너진다.
정확히는:

dot
=
최소 place-state

ㆍ
=
문맥에 따라 point-like operating mark / transition mark / 벡터씨앗으로 읽힐 수 있는 기호

이다.

3. P_place를 단순 좌표로 축소

1단계 결과문은 P_place를 단순 좌표가 아니라 다음처럼 둔다.

P_place = 놓일 수 있는 자리 / 관계가 펼쳐지는 장

또 퍼콜레이션에서는 격자 / 매질 / 통과 가능한 장, 랜덤 그래프에서는 graph field / 그래프가 놓이는 자리, 칸–칼라이에서는 finite ground set X 또는 2^X field로 대응시킨다.

따라서 P_place = 좌표값으로 줄이면 안 된다.

P_place
=
coordinate only
아님

P_place
=
place / domain / field / possible relation-space
맞음
4. ㅣ을 벽으로만 읽는 오독

ㅣ은 단순 벽이 아니다.
논문에서는 천지인과 언어-field에서 ㅣ을 다음처럼 둔다.

ㅣ = vertical axis / human-standing axis

즉 ㅣ은 세워진 축, 기준축, 인간-서기 축이다.

또 SEUNGE_FLOW_SOURCE에서는 기본 기호 의미를 다음처럼 정리한다.

ㅇ = 영역 / 장
ㅣ = 축
ㆍ = 벡터 / 입사 흐름
ㅡ = 평면
ㅎ = 회전

그리고 맞을 “영역에서 벡터가 축을 따라 움직이며 회전을 만드는 접촉”이라고 정의한다. 여기서 ㅣ은 벽이 아니라 축(axis) 이다.

따라서 ㅣ = wall로만 읽으면 Ctp의 Axis 조건이 사라진다.

5. 9ㆍ0 / 수열도형을 증명으로 과속

논문은 정수와 소수 해석을 명시적으로 “수학적 정리의 증명이 아니다”라고 제한한다.

본 논문은 수학적 증명을 주장하지 않는다.
다만 구조원리적 reading으로 다음을 보존한다.

그리고 “이 해석은 수학적 정리의 증명이 아니다. 이는 존재성, 분포, gap, 임계, 전이라는 문제 구조를 읽기 위한 구조원리적 관점”이라고 못박는다.

1단계 결과문에서도 9ㆍ0은 칸–칼라이에서 다음처럼 구조화된다.

q(F) = 9에 가까운 기대 포화자리
ㆍ = 기대와 실제 사이의 임계사이영역
p_c(F) = 0으로 넘어가는 실제 구조출현 시작자리

이것은 구조 판독이지, 그 자체로 수학적 증명 완료가 아니다.

따라서 위험지점은 이것이다.

9ㆍ0
=
임계사이 구조를 읽는 표기

9ㆍ0
≠
그 자체로 증명
6. 수학 해석과 수학 증명 혼동

논문의 한계 조항이 가장 직접적인 근거다.

논문은 다음을 scope 밖으로 둔다.

수학 난제 해결 주장
기존 물리학 이론 대체
구조원리적 reading을 수학적 증명으로 제시하는 것

또 이 framework는 conceptual framework이며, 실제 과학적 모델, 수학적 증명, 실험적 검증으로 이어지려면 별도의 정식화와 검증이 필요하다고 말한다.

따라서 Ctp 해석이 유지 가능하다는 말은:

수학적으로 해석 가능한 구조정합성

이지,

일반 수학계 형식증명 완료

가 아니다.

7. 코딩층을 일반 증명으로 오인

1단계 결과문은 3층 검산을 분리한다.

언어층 = 말로 구조가 이해되는가
수학층 = 같은 구조가 수학식으로 유지되는가
코딩층 = 같은 구조가 입력·조건·출력으로 검산되는가

그리고 닫힘 조건은 “언어에서 말한 구조 = 수학에서 압축한 구조 = 코드에서 판정한 구조”이다.

즉 코딩층은 검산층이다.
코드가 있다고 해서 그것이 일반 증명은 아니다.

1단계 결과문에 제시된 Ctp_emergence 코드도 구조출현 여부를 입력·조건·출력으로 판정하는 형태이지, 수학 정리를 증명하는 형식증명기가 아니다.

따라서 위험지점은 이것이다.

코딩층
=
실행 가능한 검산 구조

코딩층
≠
일반 수학 증명
