# 2단계 / 3회차 시작 Graph connectivity threshold



주제: giant component와 전체 연결 닫힘은 어떻게 다른가?
[2단계 3회차]

대상:
Graph connectivity threshold

중심 질문:
giant component가 출현한 상태와
그래프 전체가 하나로 연결된 상태는 어떻게 다른가?

Ctp식 질문:
관계장이 출현한 것과
전체 장이 하나로 닫힌 것은 같은가?

2단계 2회차에서는 Perfect matching을 “모든 ㆍ가 정확히 하나의 짝 link에 들어가 남김없이 보존되는 조건”으로 정리했고, Hamilton cycle·Perfect matching·Connectivity를 서로 구분해야 한다는 흐름을 이미 세웠다.

0. 3회차 핵심 결론부터
Graph connectivity threshold는
“거대한 관계장이 생겼는가?”가 아니라
“모든 ㆍ가 하나의 연결 구조 안에 들어왔는가?”
를 묻는 문제다.

Ctp식으로는 다음처럼 나뉜다.

giant component
=
관계장 출현

connectivity
=
전체 장의 연결 닫힘

즉, giant component는 중심 구조의 출현이고, connectivity는 잔여 없는 전체 연결이다.

giant component:
큰 덩어리 하나가 생김

connectivity:
모든 ㆍ가 하나의 덩어리 안에 들어감

따라서 3회차의 핵심은 이것이다.

관계장이 생긴 것과
전체 관계장이 닫힌 것은 다르다.

3. 전제조건 사전정의
ㆍ:
그래프 안에 놓인 최소 자리 / vertex / 존재씨앗

link:
두 ㆍ 사이를 잇는 연결고리 / edge

path:
link들이 이어져 한 ㆍ에서 다른 ㆍ로 도달하는 경로

component:
서로 path로 도달 가능한 ㆍ들의 부분 관계장

giant component:
전체 ㆍ 중 큰 비율을 포함하는 거대 부분 관계장

connectivity:
모든 ㆍ가 하나의 component 안에 들어간 전체 연결 닫힘

어떤 link도 갖지 못해 relation-field에 들어오지 못한 ㆍ

threshold:
connectivity가 출현했다고 판정되는 대표 좌표

임계사이영역:
고립점과 작은 component가 사라지고 전체 연결 닫힘으로 넘어가는 전이장

5. 1단계·2단계 앞 회차와의 연결
1단계 랜덤 그래프 전이
small components
→ giant component

이것은 구조장 출현이다.

2단계 1회차 Hamilton cycle
relation-field
→ cyclic closure

이것은 순환 닫힘이다.

2단계 2회차 Perfect matching
relation-field
→ pairing preservation

이것은 짝 보존이다.

2단계 3회차 Connectivity
giant component + 잔여 component
→ one connected component

이것은 전체 연결 닫힘이다.

8. 단어 구조문
Graph connectivity threshold는
랜덤 그래프 안에서 link들이 충분히 누적된 뒤,
거대한 관계장이 생기는 수준을 넘어
모든 ㆍ가 하나의 component 안에 포함되는
전체 연결 닫힘 조건을 묻는 문제다.

Ctp로 보면,
이 문제는 relation-field emergence가 total connected closure로 올라가는 순간을 판독하는 문제다.

giant component는 중심 관계장의 출현이지만,
connectivity는 잔여 ㆍ와 잔여 component가 사라져
전체가 하나의 관계장으로 닫힌 상태다.
9. 수학층

무작위 그래프:

G(n,π)

Connectivity:

G is connected iff
for every u, v ∈ V,
there exists a path from u to v.

고립점 조건:

isolated vertices = 0

랜덤 그래프의 대표 임계 스케일:

π ≈ (log n) / n

더 정밀하게는:

π = (log n + c) / n

부근에서 connectivity probability가 전이된다.

Ctp식:

C = Ctp(t, P_place, m, ?)

P_place = graph field G(n,π)

m = connectivity emergence event

t = D(giant_component_state, connected_graph_state)

? = {n, π, edge rule, independence, isolated vertex condition, connectivity criterion}

C = total_relation_field_closure
10. 코딩층
def find_components(graph):
    """
    실제 구현에서는 DFS/BFS 또는 Union-Find 사용.
    여기서는 구조형 placeholder.
    """
    return graph.get("components", [])


def isolated_vertices(graph):
    degrees = {v: 0 for v in graph["vertices"]}
    for a, b in graph["edges"]:
        degrees[a] += 1
        degrees[b] += 1
    return [v for v, deg in degrees.items() if deg == 0]


def Ctp_connectivity_threshold(graph, premise):
    iso = isolated_vertices(graph)
    components = find_components(graph)

    if iso:
        return {
            "C": "not_connected",
            "status": "before_threshold",
            "reason": "isolated_vertices_exist",
            "isolated_vertices": iso,
            "m": "connectivity_emergence_event",
            "P_place": "graph_field",
            "t": "residual_dots_outside_relation_field",
            "premise": premise,
        }

    if len(components) == 1:
        return {
            "C": "total_relation_field_closure",
            "status": "connected_graph_emerged",
            "m": "connectivity_emergence_event",
            "P_place": "graph_field",
            "t": "giant_component_to_total_connectivity",
            "premise": premise,
        }

    return {
        "C": "critical_between_region",
        "status": "no_isolated_vertices_but_multiple_components",
        "m": "connectivity_emergence_event",
        "P_place": "graph_field",
        "t": "between_isolation_removal_and_total_closure",
        "premise": premise,
    }

코드층 핵심은 세 상태다.

1. isolated vertex exists
   → 전체 연결 닫힘 불가

2. no isolated vertices but multiple components
   → 임계사이영역

3. one component
   → 전체 연결 닫힘
11. 분석

Graph connectivity threshold는 giant component 이후에 오는 더 강한 닫힘 문제다.

giant component:
관계장이 출현했다.

connectivity:
모든 ㆍ가 그 관계장 안으로 들어왔다.

그러므로 connectivity는 단순히 “크다”가 아니다.

큰 관계장 하나 + 바깥에 남은 점들
≠ connectivity

Connectivity는 잔여가 없어야 한다.

잔여점 없음
잔여 component 없음
모든 ㆍ가 하나의 relation-field 안에 있음

이것은 Ctp에서 전체 연결 닫힘이다.

12. Hamilton cycle / Perfect matching / Connectivity 비교
항목	Hamilton cycle	Perfect matching	Connectivity
핵심	순환 닫힘	짝 보존	전체 연결 닫힘
모든 ㆍ 포함	예	예	예
각 ㆍ의 조건	순환에서 2 link 사용	정확히 1 link 사용	최소 path로 전체와 연결
구조판정	복귀 + 보존	배정 + 보존	잔여 없음 + 도달 가능
Ctp 결과	cyclic closure	pairing preservation	total relation-field closure

세 문제 모두 2단계 문제지만, 닫힘의 방식이 다르다.

Hamilton cycle:
하나의 순환으로 닫힘

Perfect matching:
짝들로 분해되어 보존

Connectivity:
하나의 연결장으로 닫힘
13. 통과 / 보정 / 보류 판정
통과
Graph connectivity threshold는 Ctp 2단계에 적합하다.

이유:

1. giant component 이후 전체 닫힘을 다룬다.
2. 모든 ㆍ가 하나의 관계장에 들어오는지를 묻는다.
3. structure_judgment의 보존·반복·연결 검산과 맞는다.
보정

Connectivity를 Hamilton cycle이나 Perfect matching과 혼동하면 안 된다.

connectivity:
모든 점 사이에 path가 있음

Hamilton cycle:
모든 점을 한 번씩 지나 되돌아오는 cycle이 있음

Perfect matching:
모든 점이 정확히 하나의 짝 edge에 포함됨
보류

Connectivity threshold의 정밀 확률 극한식은 이번 회차의 목적이 아니다.

이번 목적:
Ctp 구조 대응

보류:
정밀 확률수렴식, limiting probability, isolated vertex distribution
