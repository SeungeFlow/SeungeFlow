# 2단계 / 7회차 시작 Lonely Runner Conjecture

주제: 원운동에서 거리차와 시점/지점은 어떻게 분리되는가?
[2단계 7회차]

대상:
Lonely Runner Conjecture

중심 질문:
원형 궤도 위에서 서로 다른 속도로 움직이는 존재들이 있을 때,
어떤 시점에는 한 존재가 다른 모든 존재와 충분한 거리사이를 확보하는가?

Ctp식 질문:
시점, 지점, 속도벡터, 원형자리, 거리사이를 분리했을 때,

0. 7회차 핵심 결론부터
Lonely Runner Conjecture는
“누가 혼자인가?”를 묻는 문제가 아니라,
“원형 자리 위에서 같은 시점에 모든 다른 존재와 충분한 거리사이를 확보하는가?”

5. 수학층
기본 위치식

unit circle을 R/Z로 두면, 속도 v_i를 가진 runner의 시점 τ에서의 지점은 다음과 같다.

x_i(τ) = {v_i τ}

여기서 { }는 fractional part, 즉 원형 자리장 위의 현재 지점을 나타낸다. Bohman–Holzman–Kleitman의 “Six Lonely Runners”도 1/(k+1) ≤ {v_i t} ≤ k/(k+1) 형태로 lonely condition을 쓴다.

stationary runner 방식

한 runner를 기준점 0에 고정하면, 다른 runner들의 상대속도만 보면 된다.

runner_0 = stationary at 0

x_i(τ) = {v_i τ}

runner_0가 lonely하려면:

∀ i,  ||v_i τ|| ≥ 1/(k+1)

여기서 ||x||는 원형에서 nearest integer까지의 거리, 즉 0 지점으로부터의 원형 거리다. Tao의 논문도 이 정식과 같은 unit circle·distinct speed·분리거리 표현을 사용한다.

Ctp식
C = Ctp(t, P_place, m, ?)

P_place = S¹ or (S¹)^k

m = lonely_state_event_for_runner_i

t = gap_i(τ)
  = min_j distance_S¹(x_i(τ), x_j(τ))

? = {k, speeds v_i, unit_circle_metric, τ, threshold 1/(k+1)}

C = lonely_time_exists if ∃τ such that gap_i(τ) ≥ 1/(k+1)

8. 언어층
Lonely Runner Conjecture는
서로 다른 일정 속도로 원형 트랙을 달리는 runner들이 있을 때,
각 runner가 어느 시점에는 다른 모든 runner와 충분히 떨어져 있게 되는지를 묻는다.

Ctp로 보면,
runner는 움직이는 존재씨앗이고,
unit circle은 복귀 가능한 원형 자리장이다.

시점 τ가 정해지면 각 runner의 지점 x_i(τ)가 정해진다.
외로움은 감정이 아니라,
같은 시점에서 runner i와 다른 모든 runner 사이의 거리사이가
기준거리 이상으로 확보된 상태다.

따라서 이 문제는 원형 운동에서
속도벡터가 만들어내는 지점 변화,
시점과 지점의 분리,
그리고 거리사이의 최솟값을 판독하는 문제다.
9. 코딩층
import math

def frac(x: float) -> float:
    return x - math.floor(x)


def circle_distance(a: float, b: float) -> float:
    diff = abs(frac(a) - frac(b))
    return min(diff, 1.0 - diff)


def positions_at_time(speeds, tau):
    return [frac(v * tau) for v in speeds]


def loneliness_gap_for_runner(speeds, tau, runner_index):
    positions = positions_at_time(speeds, tau)
    target = positions[runner_index]

    distances = [
        circle_distance(target, positions[j])
        for j in range(len(speeds))
        if j != runner_index
    ]

    return min(distances) if distances else 0.0


def Ctp_lonely_runner(speeds, runner_index, premise):
    """
    구조형 검산.
    실제 증명이 아니라 지정된 time grid에서 lonely state를 찾는 판정 구조.
    """
    k_plus_1 = len(speeds)
    threshold = 1.0 / k_plus_1

    for step in range(premise["time_steps"] + 1):
        tau = premise["time_start"] + step * premise["time_delta"]

        gap = loneliness_gap_for_runner(speeds, tau, runner_index)

        if gap >= threshold:
            return {
                "C": "lonely_time_exists",
                "status": "lonely_state_found",
                "m": f"runner_{runner_index}_lonely_event",
                "P_place": "unit_circle_phase_space",
                "t": "minimum_distance_gap_at_tau",
                "?": premise,
                "tau": tau,
                "gap": gap,
                "threshold": threshold,
            }

    return {
        "C": "lonely_time_not_found_within_observation_bound",
        "status": "not_confirmed",
        "m": f"runner_{runner_index}_lonely_event",
        "P_place": "unit_circle_phase_space",
        "t": "distance_gap_not_yet_above_threshold",
        "?": premise,
        "threshold": threshold,
    }

코드층 핵심은 세 상태다.

1. invalid speed configuration
   → 전제조건 실패

2. gap ≥ threshold인 τ 발견
   → lonely state 확인

3. 관측 범위 안에서 τ를 찾지 못함
   → 미판정

중요:

not_found_within_observation_bound
≠ false

증명 실패가 아니라, 지정된 관측 범위 안에서 찾지 못했다는 뜻이다.

10. 분석

Lonely Runner Conjecture는 “거리”라는 단어가 핵심이다.

여기서 거리란 단순 직선 길이가 아니다.

거리 =
원형 자리장 위에서
두 지점 사이에 생긴 최소 원호 거리

따라서 어떤 runner가 lonely하다는 것은:

다른 모든 runner와
같은 시점에
원형 자리장에서
기준거리 이상 떨어져 있음

을 뜻한다.

이 문제에서 가장 위험한 오해는 이것이다.

runner가 같은 지점에 다시 오면 같은 상태다.

아니다.

같은 지점이라도 시점이 다르면 상태가 다르다.

또 다른 오해:

서로 멀어진다는 것은 단순히 속도 차이가 있다는 뜻이다.

아니다.

속도 차이는 지점차를 만들지만,
lonely state는 특정 시점에 모든 다른 runner와 기준거리 이상 떨어지는 조건이다.

즉:

속도차
→ 지점차 생성
→ 거리사이 판독
→ lonely state 가능성
