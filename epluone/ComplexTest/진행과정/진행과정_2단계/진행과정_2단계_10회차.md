0. 10회차 핵심 결론부터
Sphere packing / covering은
“구가 몇 개 있는가?”를 묻는 문제가 아니라
“구들이 같은 자리장 안에서
서로 겹치지 않으면서 얼마나 조밀하게 놓일 수 있는가,
또는 빈틈 없이 얼마나 효율적으로 덮을 수 있는가?”
를 묻는 문제다.

Ctp식으로는 다음처럼 나뉜다.

Sphere packing
=
경계가 서로 침범하지 않는 조건 아래
밀도를 최대화하는 자리 배열 문제

Sphere covering
=
빈틈이 남지 않는 조건 아래
덮는 비용 또는 겹침을 최소화하는 자리 덮임 문제

즉 packing과 covering은 둘 다 구를 놓는 문제지만 방향이 다르다.

packing:
겹치지 말고 최대한 채워라.

covering:
비지 않게 전체를 덮어라.

3차원 구 포장에서는 Kepler conjecture가 “동일한 구들의 어떤 포장도 face-centered cubic packing의 밀도보다 클 수 없다”고 말하며, 그 밀도는 π/√18 ≈ 0.74로 주어진다. Hales의 Annals 논문은 이 정리를 Kepler conjecture의 정리문으로 제시한다.

Sphere packing

공간:

R^d

동일 반지름 r의 balls:

B(c_i, r)

비겹침 조건:

||c_i - c_j|| ≥ 2r
for i ≠ j

Packing density:

density =
limsup_{R→∞}
volume of balls inside large region / volume of large region

3차원에서는 Kepler conjecture에 의해 face-centered cubic / hexagonal close packing의 밀도 π/√18가 최적이다.
고차원에서는 8차원 E8과 24차원 Leech lattice가 각각 최적 포장임이 증명되었고, 24차원 결과는 Cohn–Kumar–Miller–Radchenko–Viazovska의 Annals 논문에 제시되어 있다.

9. 코딩층
import math
from itertools import combinations

def euclidean_distance(a, b):
    return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))


def packing_valid(centers, radius, tolerance=1e-9):
    for a, b in combinations(centers, 2):
        if euclidean_distance(a, b) < 2 * radius - tolerance:
            return False
    return True


def covering_check_sampled(centers, covering_radius, sample_points):
    """
    연속 공간 covering을 증명하지 않는다.
    유한 sample point 기준 구조형 검산이다.
    """
    uncovered = []

    for p in sample_points:
        covered = any(
            euclidean_distance(p, c) <= covering_radius
            for c in centers
        )
        if not covered:
            uncovered.append(p)

    return {
        "all_sampled_points_covered": len(uncovered) == 0,
        "uncovered_points": uncovered,
    }


def Ctp_sphere_packing_covering(config, premise):
    centers = config["centers"]
    radius = config["radius"]
    mode = config["mode"]

    if mode == "packing":
        valid = packing_valid(centers, radius)

        if valid:
            return {
                "C": "packing_structure_valid",
                "status": "non_overlap_confirmed",
                "m": "sphere_packing_structure",
                "P_place": premise["space"],
                "t": "non_overlap_vs_density_relation",
                "?": premise,
            }

        return {
            "C": "packing_structure_invalid",
            "status": "overlap_detected",
            "m": "sphere_packing_structure",
            "P_place": premise["space"],
            "t": "boundary_intrusion_detected",
            "?": premise,
        }

    if mode == "covering":
        sampled = covering_check_sampled(
            centers,
            config["covering_radius"],
            premise["sample_points"],
        )

        if sampled["all_sampled_points_covered"]:
            return {
                "C": "covering_structure_sample_confirmed",
                "status": "no_sampled_void",
                "m": "sphere_covering_structure",
                "P_place": premise["space"],
                "t": "void_to_covered_relation",
                "?": premise,
            }

        return {
            "C": "covering_structure_not_confirmed",
            "status": "sampled_void_exists",
            "m": "sphere_covering_structure",
            "P_place": premise["space"],
            "t": "uncovered_void_detected",
            "?": premise,
            "uncovered": sampled["uncovered_points"],
        }

    return {
        "C": "invalid_mode",
        "status": "failed_premise",
        "reason": "mode must be packing or covering",
        "?": premise,
    }

코드층 핵심은 네 상태다.

1. packing mode + overlap 없음
   → packing valid

2. packing mode + overlap 있음
   → 경계 침범

3. covering mode + sampled void 없음
   → sampled covering confirmed

4. covering mode + sampled void 있음
   → covering not confirmed

주의:

유한 sample covering check는 연속공간 covering 증명이 아니다.
구조형 검산이다.
10. PRO.확장 분석

Sphere packing / covering은 Ctp의 자리, 경계, 빈틈, 밀도를 가장 직접적으로 압박한다.

Packing의 핵심
구를 많이 놓고 싶다.
하지만 서로 겹치면 안 된다.

Ctp식:

밀도 증가 욕구
vs
경계 침범 금지

이 두 조건 사이가 packing의 임계사이영역이다.

너무 멀면 밀도가 낮다.
너무 가까우면 겹친다.
경계가 닿는 순간 contact structure가 생긴다.

즉 packing은:

비겹침을 유지하면서
빈틈을 최소화하는 구조

다.

Covering의 핵심
공간을 모두 덮고 싶다.
하지만 너무 많이 겹치면 낭비다.

Ctp식:

빈틈 제거
vs
겹침 낭비 최소화

이 두 조건 사이가 covering의 임계사이영역이다.

너무 작으면 빈틈이 남는다.
너무 크면 다 덮지만 낭비가 커진다.
적절한 covering radius가 필요하다.

14. 통과 / 보정 / 보류 판정
통과
Sphere packing / covering은 Ctp 2단계에 적합하다.

이유:

1. 자리와 경계를 직접 다룬다.
2. 빈틈과 밀도를 직접 다룬다.
3. packing과 covering이 서로 반대 방향의 최적화 구조를 이룬다.
4. 000_dot, 001_line, 009_vector, 014_structure_judgment, 062_place_domain_definition과 모두 연결된다.
보정

Packing과 covering을 같은 문제로 보면 안 된다.

Packing =
비겹침 조건 아래 밀도 최대화

Covering =
무빈틈 조건 아래 덮임 비용 최소화

둘은 같은 구를 쓰지만, 실패 조건이 반대다.

보류

Sphere packing / covering의 최적해 증명 자체는 이번 회차 목적이 아니다.

보류:
Kepler conjecture proof 내부
E8 / Leech lattice proof 내부
high-dimensional asymptotic bounds
covering density 최신 상한 개선
lattice vs non-lattice 최적성

이번 회차의 목적은:

packing / covering의 Ctp 구조 대응

이다.

통과.

Sphere packing / covering은
중심과 경계를 가진 구들을 공간 자리장 안에 배치할 때,
