7. 단어 구조문
Collatz conjecture는
하나의 양의 정수 상태에서 시작해
짝수면 반으로 줄이고,
홀수면 3배 후 1을 더하는 전이규칙을 반복했을 때,

Ctp로 보면,
이 문제는 하나의 상태가 반복 전이벡터를 따라 이동하면서

따라서 Collatz는 수 하나의 문제가 아니라
반복연산이 발산하지 않고 귀환 닫힘으로 들어가는가의 문제다.

9. 코딩층
def collatz_step(n):
    if n <= 0:
        raise ValueError("Collatz is defined here for positive integers only")

    if n % 2 == 0:
        return n // 2

    return 3 * n + 1


def collatz_orbit(n0, max_steps=10000):
    n = n0
    orbit = [n]

    for step in range(max_steps):
        if n == 1:
            return {
                "status": "closed_to_1_cycle",
                "orbit": orbit,
                "steps": step,
            }

        n = collatz_step(n)
        orbit.append(n)

    return {
        "status": "not_yet_closed",
        "orbit": orbit,
        "steps": max_steps,
    }


def Ctp_collatz(n0, premise):
    result = collatz_orbit(n0, max_steps=premise["max_steps"])

    if result["status"] == "closed_to_1_cycle":
        return {
            "C": "orbit_closure_to_1_cycle",
            "status": "Collatz_closed",
            "m": f"Collatz_orbit_of_{n0}",
            "P_place": "positive_integer_state_space",
            "t": "iterative_transition_difference",
            "premise": premise,
            "steps": result["steps"],
        }

    return {
        "C": "orbit_not_yet_closed",
        "status": "undecided_within_bound",
        "m": f"Collatz_orbit_of_{n0}",
        "P_place": "positive_integer_state_space",
        "t": "transition_continues_without_confirmed_closure",
        "premise": premise,
        "steps": result["steps"],
    }

11. Collatz와 Hamilton cycle의 차이
항목	Hamilton cycle	Collatz
대상	그래프 안의 모든 vertex	하나의 정수 orbit

7. 언어층
Collatz conjecture는
하나의 양의 정수에서 시작해,
짝수이면 반으로 줄이고,
홀수이면 세 배 한 뒤 하나를 더하는 전이함수를 반복했을 때,
그 상태열이 결국 1 → 4 → 2 → 1의 닫힌 순환으로 들어가는지를 묻는 문제다.

Ctp로 보면,
관측대상 m은 정수 하나가 아니라 그 정수에서 시작한 orbit 전체다.

자리 P_place는 양의 정수 상태공간이고,
차이 t는 현재 상태에서 다음 상태로 넘어가는 전이차다.

전제조건 ?는 전이함수 T, 짝홀성 판정, 양의 정수 조건, 반복 기준, 닫힘 판정 기준이다.

따라서 Collatz는 반복전이 흐름이 발산하지 않고 복귀 닫힘으로 들어가는지를 판독하는 문제다.

def collatz_step(n: int) -> int:
    if n <= 0:
        raise ValueError("n must be a positive integer")

    if n % 2 == 0:
        return n // 2

    return 3 * n + 1


def collatz_orbit(n0: int, max_steps: int = 10_000):
    if n0 <= 0:
        return {
            "status": "invalid_input",
            "reason": "n0 must be positive",
            "orbit": [],
        }

    n = n0
    orbit = [n]

    for step in range(max_steps):
        if n == 1:
            return {
                "status": "closed_to_known_cycle",
                "cycle": [1, 4, 2, 1],
                "steps": step,
                "orbit": orbit,
            }

        n = collatz_step(n)
        orbit.append(n)

    return {
        "status": "not_closed_within_bound",
        "steps": max_steps,
        "orbit": orbit,
    }


def Ctp_collatz(n0: int, premise: dict):
    result = collatz_orbit(n0, max_steps=premise["max_steps"])

    if result["status"] == "invalid_input":
        return {
            "C": "invalid_domain",
            "m": f"orbit({n0})",
            "P_place": "positive_integer_state_space",
            "t": None,
            "?": premise,
            "status": result["status"],

    }
