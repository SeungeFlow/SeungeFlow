7. 단어 구조문
Hadamard matrix existence는
±1이라는 두 relation type만을 사용해
n × n 자리장을 채울 때,
모든 행벡터가 서로 직교하고
각 행벡터의 크기가 보존되는 배열이 존재하는지를 묻는 문제다.

Ctp로 보면,
이 문제는 관계상태가 +1 / -1로 분리된 자리장에서

따라서 Hadamard matrix는
차이를 없앤 구조가 아니라,
차이를 균형으로 묶어 직교 닫힘을 이룬 구조다.
8. 수학층

Hadamard matrix:

H ∈ {+1, -1}^{n×n}

정의 조건:

HH^T = nI

행벡터 조건:

for i ≠ j:
⟨r_i, r_j⟩ = 0

자기 보존 조건:

for i = j:
⟨r_i, r_i⟩ = n

Ctp식:

C = Ctp(t, P_place, m, ?)

P_place = {+1, -1}^{n×n}

m = Hadamard matrix existence event

t = D(row_i, row_j)
  = ⟨row_i, row_j⟩

? = {n, entry_rule, orthogonality_criterion, equivalence_operations}

C = orthogonal_balance_closure

여기서 t는 단순한 수치 차가 아니다.

t =
두 row vector 사이의 자리별 relation type 차이가
내적으로 압축된 관계차

Hadamard 조건은 모든 서로 다른 row pair에 대해:

t = 0

9. 코딩층
def inner_product(row_a, row_b):
    return sum(a * b for a, b in zip(row_a, row_b))


def is_hadamard_matrix(H):
    n = len(H)

    if n == 0:
        return False

    if any(len(row) != n for row in H):
        return False

    for row in H:
        if any(x not in (+1, -1) for x in row):
            return False

    for i in range(n):
        for j in range(n):
            ip = inner_product(H[i], H[j])

            if i == j and ip != n:
                return False

            if i != j and ip != 0:
                return False

    return True


def Ctp_hadamard_existence(H, premise):
    n = len(H)

    if n not in (1, 2) and n % 4 != 0:
        return {
            "C": "Hadamard_impossible_by_order_condition",
            "status": "failed_premise",
            "reason": "order_not_1_2_or_multiple_of_4",

10. 분석

Hadamard matrix existence는 지금까지의 그래프 문제들과 성격이 다르다.

그래프 문제들은 대체로:

점과 link가 누적되어 어떤 구조가 출현하는가?

를 물었다.

Hadamard는 이렇게 묻는다.

이미 자리장이 있다.
그 자리장을 ±1로 채울 때,
모든 row 관계가 서로 직교 균형으로 닫히는가?

즉 핵심은 연결이 아니라 균형이다.

+1과 -1이 아무렇게나 섞이면 배열이다.
+1과 -1이 모든 row pair에서 균형을 이루면 Hadamard 구조다.

여기서 “차이”는 사라지는 게 아니다.

차이는 존재한다.
하지만 모든 row pair에서 그 차이가 0으로 균형 닫힘을 이룬다.

이게 Ctp적으로 중요하다.

차이를 없애는 것이 아니라,
차이를 균형 구조로 묶는다.
11. Hadamard와 Ramsey의 차이
항목	Ramsey	Hadamard
핵심	충분히 크면 패턴이 필연적으로 생김	±1 배열이 직교 균형으로 닫히는가
relation type	색 / 상태	+1 / -1
구조	unavoidable pattern	orthogonal balance closure
중심 조건	패턴 회피 불가능	모든 row pair 내적 0
Ctp 결과	필연 패턴 출현	직교 균형 닫힘

Ramsey는 “어떻게 배치해도 특정 패턴이 생긴다”는 문제다.
Hadamard는 “어떻게 배치해야 모든 관계차가 0으로 균형을 이루는가”의 문제다.
