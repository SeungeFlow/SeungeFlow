`time.state` is not a fourth slot placed beside `open.state`, `vector.state`, and `place.state`.

```text

=
the continuous condition that passes through
```


## 3. time.state as Continuous Condition

Written signs appear bounded.
Expression is continuous.

Examples:

```text
document symbols
musical notation
letters
sound signs
state labels
rendering frames
```

These appear as separated marks or bounded stages.
However, the actual expression flows continuously.

```text
written signs appear bounded
expression continues through time.state
```

Therefore, `time.state` is not an object inserted beside the signs.
It is the continuity condition through which the signs are expressed, read, and connected.

## 4. dot as Observation Cut

`dot` does not mean that time itself is a point.

```text
dot
=
an observation cut used to read a long continuous time-region
```

Korean core statement:

```text
dot은 연속된 time.state를 읽기 위해 찍는 관측 절단점이다.
```

A long time-region cannot be read all at once by the observer.
The observer places dots, symbols, letters, notes, frames, or states as readable cut points.

```text
continuous time.state
→ observation cut
→ dot
→ readable stage
```

The dot is not the flow itself.
The dot is a reading mark on the flow.

## 6. Ctp Reading

In rendering, `m`, `t`, `p`, and `?` can be stabilized as follows.

```text
m
=
existence revealed in open.state

t
=
difference / flow / direction arising in vector.state

p
=
place settled in place.state

?
=
observer criterion for reading the whole flow saturated with time.state
```

Thus:

```text
Ctp24(m, t, p, ?)
```

should not add `time.state` as a separate fourth object beside `m`, `t`, and `p`.
Instead, `time.state` is the continuous condition through which the `m → t → p` flow is read.
