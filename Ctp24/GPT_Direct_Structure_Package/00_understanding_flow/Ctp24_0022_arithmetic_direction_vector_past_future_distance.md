## 1. 곱셈과 덧셈의 차이

덧셈은 하나와 하나가 이어지는 것이다.

```text
1 + 1 = 1 + 1
```

즉, 덧셈은 두 사이영역이 관계를 맺어 이어진 하나가 되는 구조다.

곱셈은 덧셈을 덧셈만큼 배열한 것이다.

```text
곱셈 =
덧셈을 덧셈만큼 배열한 것
```

예를 들어:

```text
1년 + 1년 + 1년 ...
```

이 반복이 일정한 구조로 배열되면 곱셈이 된다.

```text
1년 × 10 =
1년 덧셈을 10개 배열한 것
```

따라서:

```text
덧셈 =
이어짐

곱셈 =
이어짐의 배열
```

---

## 2. 나눗셈과 뺄셈의 차이

뺄셈은 하나와 하나 사이의 차이를 드러낸다.

```text
C - C
```

즉, 뺄셈은 두 formed 구조 사이의 거리차, 결핍, 경계, 분리를 읽는다.

나눗셈은 뺄셈을 뺄셈만큼 배열한 것이다.

```text
나눗셈 =
뺄셈을 뺄셈만큼 배열한 것
```

예를 들어 100년을 10년 주기로 나누는 것은, 100년이라는 닫힌 경계 안에서 10년씩 차이를 잘라내며 배열하는 것이다.

```text
100년 ÷ 10 =
100년 안에서 10년 단위 차이를 배열한 것
```

따라서:

```text
뺄셈 =
차이

나눗셈 =
차이의 배열
```

---

## 3. C+1과 C-1

이 구조는 다음처럼 쓸 수 있다.

```text
C+1 = C + C
```

이것은 두 C가 관계를 맺어 다음 C+1로 드러나는 것이다.

```text
C+1 =
C와 C를 이어 펼친 것
```

반대로:

```text
C-1 = C - C
```

이것은 같은 C 안에서 차이를 빼내거나, 압축하거나, 기준점으로 되돌아가는 구조다.

```text
C-1 =
C와 C 사이의 거리차를 읽어 압축한 것
```

따라서:

```text
C+1 =
펼침

C-1 =
압축
```

---

## 4. 같은 대상을 펼쳐볼 것인가, 압축시켜 볼 것인가

사칙연산의 핵심은 같은 대상을 어떤 방향으로 볼 것인가이다.

```text
펼쳐볼 것인가?
압축시켜 볼 것인가?
```

덧셈과 곱셈은 대상을 펼친다.

```text
덧셈 =
이어 펼침

곱셈 =
이어짐을 배열하여 확장
```

뺄셈과 나눗셈은 대상을 압축한다.

```text
뺄셈 =
차이를 드러내며 줄임

나눗셈 =
차이를 배열하여 기준단위로 압축
```

즉:

```text
+ / × =
펼침 계열

- / ÷ =
압축 계열
```

---

## 5. 100년 안의 1년에도 10×10이 있다

100년을 10×10으로 볼 수 있다면, 1년 내부에도 10×10이 존재할 수 있다.

이 말은 1년을 100일로 만든다는 뜻이 아니다.

1년은 365일이다.

그러나 365일을 관측기준에 따라 10×10 구조로 잘라서 볼 수 있다.

```text
1년 =
365일

365일 =
10×10 구조로 해석 가능
```

즉, 10×10은 절대 단위가 아니라 스케일 해석 방식이다.

```text
100년의 10×10
1년의 10×10
1달의 10×10
1일의 10×10
```

모두 가능하다.

이것이 시간을 접어서 본다는 의미와도 이어진다.

```text
시간접힘 =
같은 시간장을 다른 스케일의 평면으로 다시 접어 보는 것
```

---

## 6. 순방향과 역방향

덧셈과 곱셈은 공전궤도의 순방향으로 볼 수 있다.

```text
덧셈 / 곱셈 =
순방향
양수벡터
positive
```

뺄셈과 나눗셈은 공전궤도의 역방향으로 볼 수 있다.

```text
뺄셈 / 나눗셈 =
역방향
음수벡터
negative
```

즉:

```text
+ / × =
forward vector

- / ÷ =
reverse vector
```

그러나 둘은 서로 끊어진 것이 아니다.

둘은 같은 궤도 위에서 방향이 다른 해석이다.

---

## 7. 시간은 흐르고, 우리는 반대로도 해석한다

시간은 흐른다.

시간은 이어져 있다.

```text
시간 =
흐름
+
이어짐
```

그러나 인간은 시간이 흐르는 방향으로만 해석하지 않는다.

우리는 시간이 흐르는 방향과 반대 방향으로도 시간을 해석한다.

```text
순방향 =
현재에서 미래로 흐르는 방향

역방향 =
현재에서 과거로 거슬러 읽는 방향
```

이 역방향 해석을 우리는 과거라고 부른다.

---

## 8. 과거란 무엇인가

과거는 사라진 것이 아니다.

과거는 현재시점을 기준으로, 내가 지나온 위치와 내가 현재 서 있는 위치 사이의 거리차다.

```text
과거 =
현시점을 기준으로
지나간 내가 서 있던 위치와
현재 내가 서 있는 위치 사이의 거리차
```

즉:

```text
과거 =
지나간 거리차
```

여기서 과거는 시간의 반대 방향으로 해석된 거리차다.

---

## 9. 현재시점은 기준에 따라 변한다

현재시점은 하나로 고정되지 않는다.

기준에 따라 현재시점은 달라진다.

```text
1세기 기준의 현재
1년 기준의 현재
1달 기준의 현재
1주일 기준의 현재
1일 기준의 현재
1시간 기준의 현재
1분 기준의 현재
1초 기준의 현재
```

즉, 현재시점은 절대점이 아니라 기준단위에 따라 달라지는 관측점이다.

```text
현재시점 =
관측기준에 따라 달라지는 기준점
```

---

## 10. 미래란 무엇인가

미래란 현재시점을 역방향으로 과거에 놓을 때, 시간이 흐르는 방향만큼의 거리차로 볼 수 있다.

즉, 내가 서 있던 위치에서 내가 서 있는 위치까지의 거리차를 알고 있다면, 그 거리차를 현재에서 앞으로 적용할 수 있다.

```text
미래 =
현재시점을 과거 위치에 놓고,
시간이 흐르는 방향으로 본 거리차
```

따라서 미래는 아직 오지 않은 절대공간이 아니라, 현재와 과거의 거리차를 바탕으로 순방향으로 열린 가능거리다.

```text
과거 =
내가 서 있던 위치 → 내가 서 있는 위치

미래 =
내가 서 있는 위치 → 내가 서게 될 가능 위치
```

## 12. 핵심 판정

곱셈은 덧셈을 덧셈만큼 배열한 것이다.

나눗셈은 뺄셈을 뺄셈만큼 배열한 것이다.

```text
덧셈 =
이어짐

곱셈 =
이어짐의 배열

뺄셈 =
차이

나눗셈 =
차이의 배열
```

따라서:

```text
C+1 = C + C
C-1 = C - C
```

같은 대상을 펼쳐보면 + / × 이고, 압축시켜 보면 - / ÷ 이다.

시간은 흐르고 이어져 있으며, 우리는 시간의 흐르는 방향과 반대로도 시간을 해석한다.

그 역방향 해석이 과거다.

미래는 현재시점을 과거에 놓고, 시간이 흐르는 방향만큼의 거리차로 읽을 수 있다.
