## 0. 문서의 자리

이 문서는 `Path.md`의 보류장이다.

Path.md는 C와 C, form과 form, meta와 meta, branch와 branch, Event와 Context가 어떤 사이와 차이와 6W1H 관계를 통해 C+1을 여는지 기록하는 relation path schema다.

그러나 Path를 너무 빨리 닫으면 관계가 고정되어 버린다.

따라서 `path_notes.md`는 Path를 확정하기보다, 다음 relation이 열릴 자리를 남긴다.

```text
path_notes.md =
Path 보류장
+
관계 후보장
+
다음 relation 확장 자리
```

---

## 1. Path는 닫지 않는다

Path는 고정된 선이 아니다.

Path는 관측자의 시선과 경계설정에 따라 formed 되는 relation이다.

```text
Path =
fixed route X
relation path O
```

따라서 지금 단계에서 Path를 완전히 닫지 않는다.

```text
확정:
Path는 relation path다.

보류:
어떤 C와 C를 먼저 연결할 것인가.
```

---

## 2. 현재 확정된 Path 원리

현재 확정된 Path 원리는 다음이다.

```text
1. Path는 파일경로가 아니다.
2. Path는 relation path다.
3. relation은 사이와 차이다.
4. Path는 6W1H 질문축을 가진다.
5. Path는 C+1=C+C를 핵심 구조식으로 둔다.
6. Path는 관측자의 시선이 머무는 곳에서 열린다.
7. Path는 Core와 다르다.
8. Path는 meta와 meta 사이의 relation을 읽는다.
9. Path는 Event와 Context를 연결한다.
10. Path는 branch와 branch 사이의 역할 흐름을 읽는다.
```

## 4. Path와 Event/Context 보류

Event와 Context는 Path의 중요한 적용 대상이다.

그러나 아직 첫 적용 대상을 확정하지 않는다.

후보는 다음이다.

```text

Sejong ↔ Hunminjeongeum

```

보류 이유는 다음이다.

```text

그 다음 Event/Context를 넣어야 한다.
```

따라서 지금은 Event/Context 후보를 남기고, 다음 작업에서 선택한다.

## 7. Path와 C=tp / C=(m,t,p,?) 보류

C=tp와 C=(m,t,p,?)는 모두 Path에 관여한다.

```text
C=tp =
전체 흐름식

C=(m,t,p,?) =
세부 분해·분석·검증·확인·의심 순환식
```

보류 질문:

```text
1. C=tp에서 C=(m,t,p,?)로 넘어가는 Path는 무엇인가?
2. C=(m,t,p,?)에서 다시 C=tp로 돌아오는 Path는 무엇인가?
3. 어떤 경우에는 C=tp로 읽고, 어떤 경우에는 C=(m,t,p,?)로 읽는가?
```

현재 판정:

```text
C=tp와 C=(m,t,p,?) 사이에는 왕복 Path가 있다.
그러나 구조는 단방향이고,
왕복은 관측자의 시선 이동으로 읽는다.
```
