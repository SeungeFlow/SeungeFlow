## 0. 문서의 자리

이 문서는 `Core.md`의 보류장이다.

Core.md는 C 내부의 form들이 논리적 매트릭스 구조체로 formed 되는 방식을 정의한다.

그러나 Core.md를 너무 빨리 닫으면 구조가 갇힌다.

따라서 `core_notes.md`는 Core를 확정하기보다, 다음 forming을 위해 열린 자리를 남긴다.

```text
core_notes.md =
Core 보류장
+
후보장
+
다음 확장 자리
```

---

## 1. Core는 닫지 않는다

Core는 고정표가 아니다.

Core는 작동 가능한 schema다.

```text
Core =
fixed table X
working schema O
```

따라서 지금 단계에서 Core를 완전히 닫지 않는다.

```text
확정:
Core는 inside matrix다.

보류:
어떤 form들이 24 Core가 될 것인가.
```

---

## 2. 현재 확정된 Core 원리

현재 확정된 Core 원리는 다음이다.

```text
1. Core는 목록이 아니다.
2. Core는 C 내부의 inside matrix다.
3. form은 단순 항목이 아니라 domain-field, place, relation candidate다.
4. Core는 수평배열과 수직관계를 가진다.
5. Core는 스왑행렬과 내부 행렬곱셈을 허용한다.
6. Core는 content보다 먼저 형성된다.
7. Core는 Path와 다르다.
8. Core는 meta를 지우지 않는다.
```

## 4. 24에 갇히지 않는다

24개 후보가 있다고 해서 24가 절대 고정값이 되는 것은 아니다.

```text
24 =
구조를 여는 기준수
```

24는 하루, 시간, 방향, 분할, 관측구조에서 나온 수다.

그러나 Core가 24라는 숫자에 갇히면 안 된다.

```text
24에 갇히면 =
Core가 표가 된다.

24를 기준으로 쓰면 =
Core가 구조판이 된다.
```

## 5. Core와 content의 관계

Core는 content가 아니다.

Core는 content가 놓일 자리다.

```text
Core =
content seat
```

따라서 다음 단계에서 내용을 넣을 때도 Core를 content로 채우지 않는다.

내용은 다음에서 온다.

```text
검색
AI vocab
seed_base 문서
thinking_flow
Structure_Principle/schema
BackData
ComplexTest
Event/Context 작업
```

그러나 그 내용은 Core가 세운 자리에 놓인다.

---

## 6. Core와 meta의 관계

meta는 형성과정의 흔적이다.

Core는 현재 작동하는 내부 schema다.

```text
meta =
append-only formation trace

Core =
current internal schema
```

Core는 meta를 요약하지 않는다.

Core는 meta에서 반복적으로 드러난 구조를 현재 작동 가능한 형태로 세운다.

---

## 7. Core와 Path의 관계

Core와 Path는 구분한다.

```text
Core =
inside matrix

Path =
between route
```

Core가 C 내부를 잡는다면, Path는 C와 C 사이를 연다.

Core 후보를 정할 때 Path 후보와 섞지 않는다.

그러나 Core와 Path는 서로 필요하다.

```text
Core 없는 Path =
흩어짐

Path 없는 Core =
고립
```
