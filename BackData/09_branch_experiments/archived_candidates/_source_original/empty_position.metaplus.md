# METAPLUS: empty_position

## 0. document character

`empty_position.metaplus.md`는 `empty_position.meta.md`를 대체하지 않는다.

이 문서는 `100_empty_position`의 의미를,
`000_dot`과 맞먹는 중요한 전환 자리로 다시 읽기 위한 metaplus 문서이다.

`empty_position.meta.md`는 이미 `000~099`를 하나씩 요약하고,
`101_three_dot_reading_mode_structure`로 넘어가기 전에
formed source phase와 rebuilt active phase를 직접 병합하지 않기 위해
비워 둔 전환 자리임을 정의한다.

이 metaplus 문서는 그 정의 위에서,
`100`이라는 숫자가 왜 중요한지,
왜 이것이 단순 누락 번호나 단순 summary가 아니라
2D 평면에서 3D active structure로 넘어가기 전의 dimension gate인지 정리한다.

## 5. why 100 equals dot-level importance

`000_dot`은 구조원리의 최초 자리이다.

```text
000_dot
=
field 안에서 관측대상이 놓일 수 있도록
처음 확보되는 최소 place marker
```

`100_empty_position`은 000~099가 하나의 평면장을 형성한 뒤,
그 평면장을 다시 하나의 dot처럼 압축한다.

```text
000~099
=
100 schema places

100 schema places
→
10×10 plane

10×10 plane
→
one summarized field-dot at 100
```

따라서 100은 000과 다른 방식으로 dot이다.

```text
000
=
origin dot

100
=
plane-completion dot
```

000은 시작 dot이고,
100은 평면 구조가 완성된 뒤 다음 차원을 열기 위해 비워지는 dot이다.

## 6. 2D plane and 3D living structure

승이는 말했다.

```text
하지만 우리는 3차원 구조에서 생활하고 있다.
```

이 문장이 중요하다.

`000~099`가 10×10 2D 평면구조를 이루는 구조원리를 충분히 형성했다면,
그 다음 문제는 단순히 101번째 개념을 추가하는 것이 아니다.

문제는 다음이다.

```text
2D formed schema plane
→
3D living active structure
```

우리는 2D 문서평면 안에서만 살지 않는다.

우리는 3차원 구조에서 생활하며,
관측대상은 방향, 깊이, 겹침, 내부/외부, 전후, 위아래, 시간흐름, 몸의 위치를 가진다.

따라서 101 이후 rebuilt active chain은
2D 평면의 단순 continuation이 아니라,
3D living structure로 넘어가기 위한 active opening으로 읽어야 한다.
