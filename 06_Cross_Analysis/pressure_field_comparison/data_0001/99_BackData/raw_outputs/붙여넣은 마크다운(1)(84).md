## 4. 첫 가사 진입 위치

첫 가사 진입은 악보상 **8마디 표시가 있는 줄**에서 확인된다.

해당 줄에는 첫 번째 가사 행으로 다음 구간이 배치되어 있다.

```text
누 구 의   주 제 런 가   맑 고
```

다음 줄, 즉 **12마디 표시가 있는 줄** 초반에서 이어지는 가사가 확인된다.

```text
고 운 산
```

따라서 1차 테스트 구간 후보:

```text
누구의 주제런가 맑고 고운 산
```

은 악보상 다음 범위로 확인된다.

| 구간       | 위치                     | 메모                 |
| -------- | ---------------------- | ------------------ |
| 시작       | 1페이지, 8마디 표시가 있는 줄     | 첫 가사 `누` 진입        |
| 진행       | 8마디 표시 줄 후반부           | `누구의 주제런가 맑고` 진행   |
| 종료       | 1페이지, 12마디 표시가 있는 줄 초반 | `고운 산`으로 마무리       |
| 정확한 박 단위 | 3회차에서 추출 필요            | 현재 단계에서는 위치 확인만 수행 |

## 5. 코드/화성 기호 확인

코드/화성 기호는 악보 전체에 표기되어 있다.

1차 테스트 구간 주변에서 확인되는 코드 기호는 다음과 같다.

| 위치        | 확인 코드                       | 메모                         |
| --------- | --------------------------- | -------------------------- |
| 8마디 표시 줄  | C, G7, C, Am, A             | 1차 테스트 구간 시작부 주변           |
| 12마디 표시 줄 | Dm, D, G, Dm, Am/C, E/B, Am | 1차 테스트 구간 종료 이후 이어지는 구간 포함 |

화성 기호는 존재하므로 이후 3회차 이후 표에서는 `코드` 열을 생성할 수 있다.

단, 기능화성 분석은 현재 단계에서 수행하지 않는다.

## 10. gpt.music 판정 필요 지점

| 판정 필요 지점     | 내용                                          |
| ------------ | ------------------------------------------- |
| 분석 기준 가사     | 1차 테스트는 상단 1절 가사만 사용할지 확인 필요                |
| 병렬 2절 처리     | 같은 음표에 병렬 부착된 2절 가사를 별도 데이터로 둘지 여부          |
| 첫 음절 시작 경계   | `누`의 정확한 박 위치는 3회차 추출 시 확정 필요               |
| 마지막 음절 종료 경계 | `산`의 지속/종료 처리 방식 확정 필요                      |
| 코드 기능 분석 여부  | operator는 코드명만 표기하고 기능화성 판단은 gpt.music으로 넘김 |
| 멜리스마/연장선 처리  | 한 음절이 여러 음표에 걸릴 경우 `연장` 또는 `분할` 기준 확정 필요    |

gpt.music.operator는 그리운 금강산 악보 원자료 상태를 확인했고, 1차 테스트 구간 추출 가능 여부를 gpt.music에게 보고한다.
