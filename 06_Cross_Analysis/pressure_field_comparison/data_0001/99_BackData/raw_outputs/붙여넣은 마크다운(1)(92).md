## 2. 실험 목적

이번 실험은 오감도를 비목 선율에 맞추어 노래로 만드는 작업이 아니다.

목적은 다음이다.

```text
오감도 언어 구조
+
비목 음계 / 음가 / 마디 / 코드 / 가사 대응 구조
=
구조 겹침 가능성 분석
```

즉, 오감도 시제1호의 행·숫자·조사·한자·대칭·경계 구조가 비목의 마디·구간·음가·코드·가사 병렬 구조와 어떤 방식으로 겹치거나 충돌하거나 미끄러지거나 잔류 후보를 만드는지 확인한다.

## 9. data ID 보관 후보

GitHub 공개 보관 시 작품명을 디렉토리명으로 쓰지 않는다.

```text
music_language/
└─ 06_Cross_Analysis/
   └─ pressure_field_comparison/
      └─ data_000X/
         ├─ metadata.md
         ├─ source_note.md
         ├─ case_manifest.md
         ├─ 01_Extracted_Data/
         ├─ 02_Jamo_Structure/
         ├─ 03_Alignment/
         ├─ 04_Observation/
         ├─ 05_Verification/
         ├─ 06_GptMusic_Judgment/
         └─ 99_BackData/
```

metadata 예시:

```text
data_id: data_000X
source_title_1: 오감도 시제1호
source_title_2: 비목
source_type_1: poem_text
source_type_2: song_score
analysis_type: structure_overlay_experiment
status: draft
public_directory_name: data_000X
```

보관 원칙:

```text
곡명 / 작품명 = metadata
디렉토리명 = data ID
원본 PDF / 악보 이미지 = 공개 업로드 금지
분석 파생 데이터 = 공개 가능
```

## 13. operator 결론

이번 1차 분해에서 확인된 것은 다음이다.

```text
오감도:
질주하는 반복 / 번호 반복 / 가·도 전환 / 1-2-2-1 대칭 / ㄷ형 경계

비목:
결정화되는 반복 / 구간 구조 / 1절·2절 병렬 / 계곡·깊이 / 맺힘·쌓임
```

1차 후보 압축:

```text
겹침 후보:
반복, 대칭, 병렬, 경계

충돌 후보:
질주 ↔ 잔류/결정화

미끄러짐 후보:
조사 전환 ↔ 음악 구간 전환

잔류 후보:
질주 부정 ↔ 맺힘/쌓임
```

이 결과는 실제 작곡이나 편곡이 아니라, 오감도 언어 구조와 비목 음악 구조의 겹침 가능성을 보기 위한 분해자료다.

gpt.music.operator는 오감도 시제1호의 언어 구조를 비목 음악 구조 위에 겹쳐 보는 1차 분해자료를 작성했고, 실제 작곡·편곡 없이 구조 겹침 후보와 gpt.music 판정 필요 지점을 보고한다.
