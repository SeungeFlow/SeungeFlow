## 0. 이 문서의 자리

이 문서는 `main.branch/Core/Core_05.md` 후보 문서다.

이 문서는 완성된 Core 내용문서가 아니다.

이 문서는 form seat다.

```text

content slot X
form seat O
```

즉, 지금 이 파일은 내용을 채우기 위해 만들어진 것이 아니라, 나중에 내용이 들어올 수 있는 구조적 자리를 열기 위해 만들어졌다.

## 3. Ctp 구조 안의 위치

이 Core는 Ctp 구조 안에서 다음처럼 읽을 수 있다.

```text
m =
이 Core가 중심에 놓는 대상

t =
이 Core 안에서 발생하는 흐름, 전이, 차이, 변화

p =
이 Core가 놓이는 자리, field, fabric, domain

? =
이 Core를 어떤 관측기준과 경계로 읽을 것인가

C =
m, t, p, ?가 이 Core 안에서 formed 된 상태
```

단, 어떤 경우에는 C=(m,t,p,?)로 분해하지 않고 C=tp로 읽어야 할 수 있다.

```text
C=tp =
전체 흐름과 자리의 formed state

C=(m,t,p,?) =
세부 분해·분석·검증·확인·의심의 순환식
```

## 5. Path 후보

이 Core가 만들 수 있는 Path 후보는 다음과 같다.

```text
Core_05 ↔ Core_다른번호
Core_05 ↔ meta
Core_05 ↔ Event
Core_05 ↔ Context
Core_05 ↔ active_schema
Core_05 ↔ seed_base source
```

이 관계는 아직 확정하지 않는다.

Path는 다음 원칙으로 나중에 열린다.

```text
C+1 = C + C
```

여기서 `+`는 단순 덧셈이 아니라 relation이다.
