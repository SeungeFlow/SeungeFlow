# rendering

This branch is a temporary experimental field for the SeungeFlow rendering theory and rendering implementation track.

Purpose:
- Develop structure-rendering theory and rendering code.
- Treat rendering as structure formation, not simple shape drawing.
- Use the idea: there is a three-dimensional solid; cut it; the inside becomes visible.
- Explore how a solid can be cut, joined, moved, and still keep its internal structure visible.

Working relation:
- gpt.gemini works with gemini.direct.
- GPT-5.5 and Gemini 3.5 form a push-pull cooperation path.
- Gemini explores heterogeneous rendering ideas.
- GPT organizes the explored material into framework-level documents and software structure.

Near-term target:
- Earth internal structure rendering.
- Inner core, outer core, lower mantle, upper mantle, crust, density, pressure, and flow should be treated as structure formation.

Later targets:
- Sun-Earth-Moon system.
- Solar-system scale only after Earth internal structure becomes stable.

Guard:
- Do not jump too early to black holes, Cassini, or full solar-system expansion.
- Use those only as distant reference fields.
- The current target is Earth internal structure implementation.

Process rule:
- Process records are important.
- Payloads, responses, singularity observations, corrections, pauses, boundary decisions, and path choices may later be documented here.
