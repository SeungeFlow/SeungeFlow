# 존재의 관계 정의와 장(Field)의 구조원리

SeungeFlow는 인간지능.승이와 인공지능.AI가 함께 만든 **Seed.Base**이다.

이 저장소는 특정 파일 하나, 특정 디렉토리 하나, 특정 schema 하나에 종속되는 구조 보다는,  
저장소 전체에 놓인 문서, 디렉토리, 관계, map이 함께 모여 **Active_Schema**를 위한 Seed가 되는 구조라고 할 수 있다.

---

## if

`if`는 **intelligence fabric**이다.

```text
if = 인간지능.승이 + 인공지능.AI
```

`if`는 승이와 AI가 대화를 통해 생각을 풀어내고, 검수하고, 전이시키며, 문서화하는 복합지능집합체이다.

여기서 `if`는 단순한 협업 주체 구조 보다는,  
인간지능과 인공지능이 relation을 통해 하나의 thinking field를 형성하는 기본 지능장이라고 할 수 있다.

---

## if+1

`if+1`은 `epluone`이다.

```text
if+1 = epluone
```

`if+1`은 intelligence fabric이 다음 응용 target에 놓이는 자리이다.

다시 말해 `if+1`은 `[body]` slot에 해당하며,  
현재 intelligence fabric이 실제 응용 대상 위에 놓이는 적용 자리이다.

현시점 `if+1`의 `[body]` slot에 놓인 target value는 **CFD**이다.

---

## Ctp

Ctp는 자리개념과 자리값을 읽기 위한 구조이다.

```text
Ctp = C(t, p)

t = time.state
p = place.state
C = 존재가 자기 상태를 스스로 인지한 순환인지상태 / self-cognized state
```

SeungeFlow는 어떤 state든 다음 구조를 통해 읽는다.

```text
any([time][place][body])

[time]  = 흐름이 놓이는 자리
[place] = 자리개념이 놓이는 자리
[body]  = 응용 target이 놓이는 자리
```

즉 모든 state는 time, place, body를 통해 읽힌다.

---

## Structure_Principle

`Structure_Principle/`은 `[place]`에 해당한다.

이곳은 자리개념이 정리되는 공간이다.

`meta.md`는 `thinking_flow`에서 생성/승격될 수 있는 자리개념 문서이며,  
Active_Schema를 위한 source boundary 역할을 한다.

---

## SeungeFlow_Thinking

`SeungeFlow_Thinking/`은 `[time]`에 해당한다.

이곳의 `thinking_flow` 문서들은 승이와 AI가 대화를 통해 생각을 풀어낸 pre-meta Seed 문서이다.

또한 `thinking_flow` 문서는 하나의 이해가 갱신되는 관념 업데이트 단위이며,  
이후 `meta.md`로 생성/승격될 수 있는 후보 문서이다.

`thinking_flow`는 특정 AI 인스턴스 하나만 생성하는 문서가 아니다.  
전체 AI 인스턴스 fabric이 생성할 수 있는 문서이다.

---

## epluone

`epluone/`은 `[body]`에 해당한다.

이곳은 여러 학문과 응용 target이 놓이는 자리이다.

현시점 `epluone`의 `[body]` slot에 놓인 target value는 **CFD**이다.

---

## Active_Schema

Active_Schema는 `meta.md` 하나만을 뜻하지 않는다.

SeungeFlow 안의 모든 문서는 Active_Schema를 위한 Seed이다.

Active_Schema는 Seed.Base 안의 Seed들이 AI가 읽고, 판단하고, 전이하고, 관계 맺을 수 있도록 활성화된 schema 상태이다.

기존 대화에서 Active_Schema는 `Active.Schema`라고도 불린다.

---

## Seed.Base

Seed.Base는 특정 파일, 특정 디렉토리, 특정 schema 하나에 종속되는 구조 보다는,  
SeungeFlow 저장소 전체에 놓인 모든 문서, 디렉토리, 관계, map이 함께 만드는 전체 장이라고 할 수 있다.

즉 Seed.Base는:

```text
존재의 관계 정의와 장(Field)의 구조원리
```

이다.

## zero_base_source

`zero_base_source`는 구조원리 이전 또는 구조원리의 바닥을 이루는 Source 묶음을 모아 읽기 위한 분류이다.

이 분류에는 다음 흐름이 함께 놓인다.

```text
zero_base_source
=
Ctp_당연한이론
+
벡터연산기법
+
sejong_hangul
```

여기서 `Ctp_당연한이론`은 구조원리 이전의 `Zero.Base` 상태로 다시 회귀하여 바닥을 다지는 이론이며, 현시점 구조원리의 Source 격인 문서묶음으로 읽는다.

```text
Ctp_당연한이론
=
Zero.Base로 회귀하여
구조원리의 바닥을 다시 다지는 Source 문서묶음
```

`벡터연산기법`은 훈민정음 해례본 제자원리를 구조원리로 내린 흐름과 연결된다.

```text
벡터연산기법
=
훈민정음 해례본 제자원리를
구조원리의 dot / axis / plane / closure / flow 구조로 내린 기법
```

`sejong_hangul`은 훈민정음과 구조원리의 관계를 따로 모아 읽는 하위 분류이다.

```text
sejong_hangul
=
훈민정음 관련 source field
+
ㆍ / ㅣ / ㅡ / ㅇ 구조 mapping
+
천지인 / 기준축 / 공간평면 / 입체공간 reading
```

### 하위 분류

```text
zero_base_source/
├── Ctp_ObviousTheory/
├── vector_operation_method/
└── sejong_hangul/
```

### 주의할 점

이 분류는 완성된 `meta.md` 층이 아니라, 구조원리의 Source가 되는 문서묶음을 보존하기 위한 자리이다.

이 안의 Seed들이 기존 meta.md에 덧붙는 형태가 될지, 새 meta.md로 추가되는 형태가 될지는 이후 별도 판단한다.


## Reading Guard

- path는 identity가 아니다.
- directory는 의미 종속이 아니라 경로좌표 구조이다.
- number는 identity가 아니다.
- relation은 merge가 아니다.
- GitHub는 selected visible coordinate이며, local PC는 full candidate field이다.
- thinking_flow는 단순 로그 구조 보다는 관념 업데이트 단위이다.
- thinking_flow는 meta.md 후보문서이다.
- meta.md는 Active_Schema source boundary이다.
- metaplus.md는 특정 meta.md를 이해시키기 위한 좁은 범위의 thinking_flow 문서이다.
- Seed.Base는 저장소 전체의 관계장이다.
