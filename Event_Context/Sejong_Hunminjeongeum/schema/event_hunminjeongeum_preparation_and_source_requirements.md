## 0. 이 문서의 자리

이 문서는 `Event_Hunminjeongeum.md` 본문이 아니다.

이 문서는 `Event_Hunminjeongeum.md`를 작성하기 전에 필요한 source 조건, 작성 경계, 금지사항, 관측 기준을 정리하는 준비문서다.

```text
이 문서 =
Event_Hunminjeongeum.md 작성 준비문서

아직 아님 =
Event_Hunminjeongeum.md 본문
```

`Context_Sejong.md`는 Sejong을 존재 형성장으로 읽는 문서다.

`Event_Hunminjeongeum.md`는 Hunminjeongeum을 formed structure로 읽는 문서다.

```text
Context_Sejong.md =
Sejong Context_C

Event_Hunminjeongeum.md =
Hunminjeongeum Event_C

Path_Sejong_Hunminjeongeum.md =
Context_Sejong_C + Event_Hunminjeongeum_C = C+1
```

따라서 이 문서는 Hunminjeongeum을 단순 업적, 문자 이름, 책 이름, 현대 한글 설명으로 바로 평탄화하지 않도록 준비한다.

## 1. Event_Hunminjeongeum.md의 목적

`Event_Hunminjeongeum.md`의 목적은 훈민정음 해설문을 쓰는 것이 아니다.

```text
Event_Hunminjeongeum.md ≠ 훈민정음 일반 설명
Event_Hunminjeongeum.md ≠ 한글 소개문
Event_Hunminjeongeum.md ≠ 업적 요약
Event_Hunminjeongeum.md ≠ 제자원리 해설문 only
```

목적은 다음이다.

```text
Event_Hunminjeongeum.md =
Hunminjeongeum이 어떤 t와 p와 ? 안에서
Event_C로 formed 되었는지 읽는 문서
```

즉, Hunminjeongeum을 하나의 formed structure로 본다.

```text
Event_Hunminjeongeum_C =
문자체계 / 책 / 제자원리 / 언어장 / 국가운영장 조건 속에서
formed 된 구조 형성물
```

이 판정은 아직 초안이며, source 확인 후 조정해야 한다.

## 2. Event_Hunminjeongeum에서 m

Event_Hunminjeongeum의 중심 m은 Hunminjeongeum이다.

그러나 여기서 Hunminjeongeum_m은 하나로 고정되지 않는다.

```text
m =
Hunminjeongeum
+
새 문자체계
+
훈민정음이라는 이름의 문자
+
훈민정음이라는 책
+
해례본의 설명 구조
+
언어 / 문자 / 국가운영장에 formed 된 구조
```

이 m은 다음 층위를 분리해야 한다.

```text
1. 새로 만든 문자체계로서의 훈민정음
2. 책 이름으로서의 훈민정음
3. 해례본의 설명 구조
4. 언해본 / 해설 계열
5. 후대의 한글 / 현대 자모 체계
```

이 층위들을 섞으면 Event가 오독된다.

## 3. Event_Hunminjeongeum에서 t

Event_Hunminjeongeum의 t는 창제일 또는 반포일 하나가 아니다.

```text
t ≠ 날짜 하나
```

t는 형성흐름이다.

```text
t =
창제 이전 조건
+
창제 표식
+
해례 / 반포 / 완성 표식
+
문자체계가 formed 되는 흐름
+
후대 확산 흐름
```

확인할 t 후보:

```text
1. 창제 이전 언어장 / 문자장 조건
2. 1443년 창제 기록
3. 1446년 완성 / 반포 관련 기록
4. 해례본의 작성과 설명 구조
5. 후대 한글 명칭과 체계로 이어지는 전이
```

날짜는 표식이고, t는 흐름이다.

```text
date =
marker

t =
formation flow
```

## 4. Event_Hunminjeongeum에서 p

Event_Hunminjeongeum의 p는 단순 장소가 아니다.

```text
p ≠ location only
```

p는 Event가 formed 된 자리장이다.

```text
p =
language field
+
writing field
+
state-operation field
+
scholarly field
+
document field
+
phonological / graphic system field
```

확인할 p 후보:

```text
1. 조선의 언어장
2. 한자 / 한문 중심 문자장
3. 백성의 문자 접근성 문제
4. 국가운영과 문서장
5. 집현전 / 학문장
6. 음운 인식과 문자 설계의 장
7. 해례본이라는 문서장
```

p를 단순 장소로 읽으면 Event가 평면화된다.

## 5. Event_Hunminjeongeum에서 ?

Event_Hunminjeongeum의 ?는 관측기준이다.

```text
? =
6W1H
+
관측자의 시선
+
경계설정
+
해석조건
+
Event 판정 조건
```

Event_Hunminjeongeum에서 ?는 다음을 묻는다.

```text
Who =
누가 Hunminjeongeum Event를 보는가?

What =
무엇을 Event의 중심 m으로 둘 것인가?
문자체계인가, 책인가, 창제 사건인가, 해례 구조인가?

When =
어디서부터 어디까지를 Event의 t로 볼 것인가?

Where =
어디까지를 Event의 p로 볼 것인가?

Why =
왜 Hunminjeongeum을 formed structure로 볼 수 있는가?

Which =
Context_Sejong에서 온 조건 중 무엇을 Event에 연결할 것인가?

How =
어떻게 Context_Sejong_C와 relation을 맺어 C+1을 여는가?
```

`?`는 바깥에만 있지 않다.

```text
? ∩ m 가능
? ∩ t 가능
? ∩ p 가능
? ∩ (m,t,p) 가능
```

훈민정음의 제자원리 자체가 Event_m의 일부이면서, 동시에 Event를 판정하는 `?`가 될 수 있다.

## 6. source requirement 전체 원칙

Event_Hunminjeongeum.md 작성에는 source가 필요하다.

source 사용 원칙은 다음이다.

```text
1. 원문 기록과 구조해석을 구분한다.
2. 창제 기록과 반포 / 완성 기록을 구분한다.
3. 문자체계와 책 이름과 후대 한글을 구분한다.
4. 해례본 원문과 현대 해설을 구분한다.
5. source 없는 역사 단정을 하지 않는다.
6. 숫자패턴을 relation 근거로 사용하지 않는다.
7. Ctp24 구조어를 역사 사실로 둔갑시키지 않는다.
8. Context_Sejong과 Event_Hunminjeongeum을 섞지 않는다.
```

Event_Hunminjeongeum.md 작성에 필요한 source 범주는 다음이다.

```text

B. 훈민정음 해례본 원문 / 원문 이미지 / 해제

```

각 source 범주는 Event의 m, t, p, ? 중 하나 이상에 연결된다.

## 20. Event_Hunminjeongeum.md 작성 전 금지사항

작성 전 금지사항:

```text
1. 훈민정음을 일반 한글 소개문으로 쓰지 않는다.
2. 훈민정음을 세종 업적 설명으로만 쓰지 않는다.
3. 해례본과 언해본을 혼동하지 않는다.
4. 창제 기록과 반포 / 완성 기록을 혼동하지 않는다.
5. 현대 자모 24와 Ctp24를 연결하지 않는다.
6. 28자 / 24자 숫자패턴을 relation 근거로 쓰지 않는다.
7. source 없는 원문 해석을 하지 않는다.
8. Context_Sejong의 조건장을 Event 안에 모두 흡수하지 않는다.
9. Path 없이 C+1을 선언하지 않는다.
10. 제자원리를 SeungeFlow 구조어로 무리하게 번역하지 않는다.
```

