## 2. Context_Sejong.md의 목적

`Context_Sejong.md`의 목적은 세종대왕의 전기를 요약하는 것이 아니다.

```text
Context_Sejong.md ≠ 위인전 요약
Context_Sejong.md ≠ 업적 목록
Context_Sejong.md ≠ 연대기 정리
```

목적은 다음이다.

```text
Context_Sejong.md =
Sejong이 어떤 t와 p와 ? 안에서
Context_C로 formed 되었는지 읽는 문서
```

즉, 세종을 하나의 인물 정보로 보는 것이 아니라, 조선 초기의 정치장, 학문장, 언어장, 문자장, 국가운영장 안에서 formed 된 존재 형성장으로 읽는다.

```text
Context_Sejong_C =
Sejong이
조선 초기 국가운영장과 학문장과 언어장 안에서
특정 관측기준 ?에 의해
formed 된 존재 형성장
```

이 판정은 아직 초안이며, source 확인 후 조정해야 한다.

## 3. Context_Sejong의 m

Context_Sejong에서 m은 세종이다.

그러나 m은 단순 인물명이 아니다.

```text
m =
Sejong
+
King Sejong
+
Yi Do
+
조선의 왕으로 formed 된 존재
+
언어장과 국가운영장에 놓인 중심 존재
```

m을 단일 이름으로 닫으면 안 된다.

세종은 다음 여러 층위로 읽힐 수 있다.

```text
왕
정치적 중심 존재
학문장 운영자
언어정책의 중심 존재
제도 형성의 중심 존재
후대 문자문화의 기준점
```

이 층위들은 source로 확인되어야 한다.

## 4. Context_Sejong의 t

Context_Sejong의 t는 세종의 연대기만이 아니다.

```text
t ≠ chronology only
```

t는 세종이 formed 되기까지의 시간흐름과 전이다.

확인할 t 후보:

```text
1. 조선 초기 국가 형성의 흐름
2. 태종에서 세종으로 이어지는 왕권 / 제도 흐름
3. 세종 개인의 성장과 학습 흐름
4. 집현전과 학문장 형성 흐름
5. 문자정책과 언어문제의 누적 흐름
6. 훈민정음 창제 전후의 전이 흐름
7. 후대 문자문화와 국가운영에 남은 영향 흐름
```

t는 날짜와 연도로 표시될 수 있지만, 날짜 자체가 t는 아니다.

날짜는 표식이고, t는 흐름이다.

## 5. Context_Sejong의 p

Context_Sejong의 p는 장소만이 아니다.

```text
p ≠ location only
```

p는 세종이 놓인 자리장이다.

확인할 p 후보:

```text
1. 조선 초기 국가운영장
2. 왕권과 제도장의 구조
3. 유교 정치질서와 학문장
4. 집현전이라는 학술장
5. 언어장 / 문자장 / 백성의 문자 접근성 문제
6. 동아시아 문자문화장
7. 행정과 법제의 문서장
8. 과학기술 / 천문 / 음악 / 제도 정비의 장
```

p는 세종이라는 m이 놓인 field / fabric / domain이다.

p를 위치로만 읽으면 Context가 평면화된다.

## 6. Context_Sejong의 ?

Context_Sejong의 ?는 관측기준이다.

```text
? =
6W1H
+
관측자의 시선
+
경계설정
+
해석조건
```

Context_Sejong에서 ?는 다음을 묻는다.

```text
Who =
누가 세종 Context를 보는가?

What =
무엇을 Context의 중심 m으로 둘 것인가?

When =
언제부터 언제까지를 세종 Context로 볼 것인가?

Where =
어디까지를 세종의 p로 볼 것인가?

Why =
왜 이 조건이 훈민정음 Event와 relation을 맺는가?

Which =
어느 조건과 어느 형성물 사이를 볼 것인가?

How =
이 Context가 Hunminjeongeum Event와 어떻게 연결되는가?
```

?는 바깥에만 있지 않다.

```text
? ∩ m 가능
? ∩ t 가능
? ∩ p 가능
? ∩ (m,t,p) 가능
```

예를 들어 `문자정책`은 p의 일부이면서, 동시에 Hunminjeongeum Event를 읽는 ?가 될 수 있다.

## 7. source requirement 전체 원칙

Context_Sejong.md 작성에는 source가 필요하다.

그러나 source를 가져와도 바로 구조해석으로 단정하면 안 된다.

source 사용 원칙:

```text
1. 역사 사실과 구조해석을 구분한다.
2. source 없는 단정을 하지 않는다.
3. source를 Ctp24 구조어로 강제 번역하지 않는다.
4. 숫자패턴을 relation 근거로 사용하지 않는다.
5. 현대 해석과 당시의 맥락을 구분한다.
6. Context와 Event를 섞지 않는다.
7. source를 직접 인용할 때는 정확히 표시한다.
8. AI vocab으로 채운 내용과 검증된 source를 구분한다.
```
