## 0. 이 문서의 자리

이 문서는 `Path_Sejong_Hunminjeongeum.md`의 초안이다.

이 문서는 세종대왕과 훈민정음을 단순히 연결하는 문서가 아니다.

이 문서는 `Context_Sejong_C`와 `Event_Hunminjeongeum_C` 사이에서 어떤 relation이 formed 되고, 그 relation이 어떤 `C+1` 후보를 여는지 관측하기 위한 Path 문서다.

```text
Path_Sejong_Hunminjeongeum.md
=
Context_Sejong_C
+
Event_Hunminjeongeum_C
가 relation을 맺어
C+1 후보를 여는 구조연산 경로
```

중심식은 다음이다.

```text
Context_Sejong_C + Event_Hunminjeongeum_C = C+1
```

여기서 `+`는 단순 덧셈이 아니다.

```text
+ =
relation
```

따라서 이 문서는 다음을 다룬다.

```text
Context와 Event를 병합하지 않는다.
Context와 Event를 무관하게 두지도 않는다.
둘 사이의 사이와 차이를 읽는다.
그 사이와 차이에서 임계사이영역과 전이를 찾는다.
C+1은 마지막에 조심스럽게 판정한다.
```

## 2. Context_Sejong_C 확인

먼저 `Context_Sejong_C`를 확인한다.

`Context_Sejong_C`는 세종대왕 위인전이 아니다.

```text
Context_Sejong_C =
Sejong이
조선 초기 국가운영장,
학문장,
언어장 / 문자장,
왕권과 제도장의 흐름 속에서
특정 관측기준 ?에 의해
formed 된 존재 형성장
```

[Provisional Judgment]

이 판정은 아직 source 검산 전의 임시 판정이다.

Context_Sejong_C는 다음 구조항을 가진다.

```text
m =
Sejong / Yi Do / 왕으로 formed 된 존재

t =
조선 초기 국가운영과 학문장, 문자장 문제의 시간흐름

p =
국가운영장, 학문장, 언어장, 문자장, 문서장

? =
Sejong을 Hunminjeongeum Event와 relation 가능한 Context로 보는 관측기준
```

[Do Not Merge]

Context_Sejong_C는 Event_Hunminjeongeum_C를 포함하지 않는다.

즉, Hunminjeongeum 자체의 문자체계, 해례본 구조, 제자원리는 Context 문서에서 완성하지 않는다.

## 3. Event_Hunminjeongeum_C 확인

다음으로 `Event_Hunminjeongeum_C`를 확인한다.

`Event_Hunminjeongeum_C`는 한글 일반 소개문이 아니다.

```text
Event_Hunminjeongeum_C =

새 문자체계와 그 설명 구조로
formed 된 구조 형성물
```

[Provisional Judgment]

이 판정 역시 source 검산 전의 임시 판정이다.

Event_Hunminjeongeum_C는 다음 구조항을 가진다.

```text
m =
Hunminjeongeum / 새 문자체계 / 책 / 해례본 설명 구조

t =
창제 이전 조건, 창제 표식, 완성 / 반포 표식, 해례 구조의 형성 흐름

p =
언어장, 문자장, 국가운영장, 학문장, 문서장, 음운·문자 설계장

? =
Hunminjeongeum을 formed structure로 보는 관측기준
```

[Do Not Merge]

Event_Hunminjeongeum_C는 Context_Sejong_C의 단순 결과로 낮추지 않는다.

Event는 Context와 relation을 맺지만, Event 자체도 하나의 C다.

이 Path의 relation 질문은 다음이다.

```text
Who =
누가 이 relation을 보는가?

What =

When =
이 relation은 어떤 시간흐름 속에서 열리는가?

Where =
이 relation은 어떤 자리장 p에서 드러나는가?

Why =
왜 Sejong Context와 Hunminjeongeum Event는 relation을 맺는가?

Which =
Context 조건 중 무엇이 Event의 formed state와 관계하는가?

How =
이 relation은 어떤 C+1 후보를 여는가?
```

이 질문에 답하지 않고 C+1을 선언하지 않는다.

Path는 질문을 통해 열려야 한다.

## 5. ?의 위치

이 Path에서 `?`는 고정된 외부 질문표가 아니다.

```text
? =
6W1H
+
관측자의 시선
+
경계설정
+
해석조건
```

`?`는 여러 위치에 놓일 수 있다.

```text
? ∩ Context_Sejong_C
? ∩ Event_Hunminjeongeum_C
? ∩ between-field
? ∩ C+1 후보
```

이 Path의 1차 `?` 후보는 다음이다.

```text
? =
Sejong Context와 Hunminjeongeum Event를
존재 형성장과 구조 형성물로 분리한 뒤,
둘 사이의 relation을 관측하려는 기준
```

즉, 이 문서의 ?는 “세종이 훈민정음을 만들었다”라는 단순 연결을 벗어나, `Context_C + Event_C = C+1`의 구조연산을 보려는 관측기준이다.

## 7. 차이

사이에서 차이가 드러난다.

Context_Sejong_C와 Event_Hunminjeongeum_C 사이의 차이 후보는 다음이다.

```text
1. 존재 형성장과 구조 형성물의 차이
2. Sejong이라는 m과 Hunminjeongeum이라는 m의 차이
3. 국가운영장의 조건과 문자체계 formed state의 차이
4. 언어 문제의 누적과 문자체계 창제의 차이
5. 창제 표식과 완성 / 반포 표식의 차이
6. 원문 기록과 구조해석의 차이
```

[Structure Interpretation]

차이는 단순 다름이 아니다.

```text
차이 =
사이영역과 사이영역 사이의 거리차
```

이 차이가 있어야 relation이 열린다.

차이가 없으면 둘은 병합된다.

차이가 relation으로 읽히지 않으면 둘은 무관해진다.

Path는 이 차이를 relation으로 읽는다.

## 10. 극한임계전이

극한임계전이는 임계사이영역을 지나 새로운 formed state가 열리는 전이다.

이 Path에서 극한임계전이 후보는 다음 사이에 놓인다.

```text

→ 창제 표식
→ 해례 / 완성 / 반포 표식
→ 문자체계의 formed state
```

[Hold Judgment]

특정 날짜를 곧바로 극한임계전이로 확정하지 않는다.

```text

marker

transition =
structural change
```

따라서 세종실록의 창제 표식과 완성 / 반포 표식은 중요한 marker지만, 극한임계전이 자체인지는 Path에서 더 검산해야 한다.

## 11. 9dot0 판정

이 Path는 9dot0 구조로 읽을 수 있다.

```text
9 dot 0
```

현재 임시 판정은 다음이다.

```text
9 =
기존 언어장 / 문자장 / 문서장의 한계 또는 끝점 후보

dot =
Context_Sejong_C와 Event_Hunminjeongeum_C 사이의 임계사이영역

0 =
Hunminjeongeum Event가 새 formed state로 드러나는 시작점 후보
```

[Provisional Judgment]

이것은 임시 판정이다.

특히 `9`를 “한자 / 한문”으로 단순화하면 안 된다.

그것은 너무 거친 판정이다.

더 정확히는 다음처럼 둔다.

```text
9 =
기존 문자장과 국가운영장 사이에서
새 문자체계 필요조건이 압축되는 끝점 후보
```

dot은 다음이다.

```text

언어장 / 문자장 / 국가운영장 / 학문장의 압력이
새 문자체계 formed state로 전이될 수 있는 임계사이영역
```

```text
0 =
Hunminjeongeum Event_C가 시작점으로 드러나는 상태 후보
```

## 14. 숫자패턴 guard

이 Path에서는 숫자패턴을 relation 근거로 사용하지 않는다.

특히 다음을 직접 연결하지 않는다.

```text
훈민정음 현대 자모 24
Ctp24
24절기
24시간
```

또한 다음도 분리한다.

```text
실록의 언문 28자 표식
현대 자모 24
옛자음 포함 체계
천지인 해석
Ctp24 form seat
```

[Number Pattern Guard]

숫자 유사성은 관찰 시작점일 수 있지만, relation의 증거가 아니다.

```text
정수패턴 =
관측 후보

정수패턴 ≠ relation 증거
```
