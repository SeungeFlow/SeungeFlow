## 1. 검산 원칙

Path source verification의 원칙은 다음이다.

```text
1. source fact와 structure interpretation을 분리한다.
2. Context와 Event를 병합하지 않는다.
3. source가 지지하는 것은 marker / condition / layer로 표시한다.
4. 임계사이영역은 source fact가 아니라 structure interpretation임을 표시한다.
5. 극한임계전이는 날짜와 동일시하지 않는다.
6. C+1은 final judgment가 아니라 provisional candidate로 둔다.
7. 숫자패턴은 relation 근거로 쓰지 않는다.
```

```text
3. 새로 28자를 만들어 사람들이 쉽게 익혀 날마다 쓰는 데 편하게 하려 했다는 취지가 제시된다.
```

Path mapping:

```text
maps_to =
between-field
+
difference
+
Context_Sejong_p
+
Event_Hunminjeongeum_t 후보
```

검산 판정:

```text
P-SH-01은
Context_Sejong_C와 Event_Hunminjeongeum_C 사이의
핵심 between-field source anchor로 적절하다.
```

구조해석:

```text
말과 글의 불일치 =
Context 쪽 언어장 / 문자장 압력

새 문자 창제 의도 =
Event 쪽 formed structure로 향하는 전이 후보
```

## 13. C+1 후보 검산

기존 C+1 후보:

```text
조선의 국가운영장과 언어장 / 문자장이
Hunminjeongeum이라는 formed structure를 통해
새로운 문자체계 field로 전이된 상태
```

source verification 후 더 안전한 문장:

```text
C+1 후보 =
조선의 언어장 / 문자장 / 국가운영장 안에서
말과 글의 불일치와 문자 접근성 문제가

```

판정:

```text
C+1 =
provisional candidate
final judgment 아님
```

이 문장은 이전보다 안전하다.

이유:

```text
1. “전이된 상태”보다 “전이될 수 있는 조건을 연 상태”로 표현했다.
2. 즉시 문자생활 혁신으로 단정하지 않았다.
3. source fact와 구조해석을 분리했다.
```
